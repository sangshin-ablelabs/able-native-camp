1 첫 대화 — Claude Code 처음 만나기

▼

Claude Code는 터미널에서 실행하는 AI 도구야. 채팅창이 아니라 터미널이야.

왜 터미널이야?

  • 파일을 직접 읽고 쓸 수 있어
  • 명령어를 실행할 수 있어
  • 내 컴퓨터의 모든 도구에 접근할 수 있어

💡 채팅 AI는 "대화만" 해. Claude Code는 "대화하면서 실행"해.

실행 방법

# 터미널을 열고 이것만 치면 돼
claude

이것만 치면 돼. Claude가 인사해.

첫 대화 예시

아무거나 시켜봐:

  • "안녕, 나는 [이름]이야. ABLE Labs [팀]에서 일해"
  • "오늘 날짜 알려줘"
  • "이 폴더에 어떤 파일이 있는지 보여줘"
  1. 1
    터미널을 열어 (아까 배운 방법!)
  2. 2
    claude 를 입력해
  3. 3
    아무 말이나 걸어봐 — 자기소개, 간단한 질문, 뭐든 좋아
  4. 4
    3번 이상 대화를 주고받아봐

💡 처음엔 어색해도 괜찮아. "오늘 점심 뭐 먹을까?" 같은 가벼운 것도 좋아.

Claude Code와 일반 ChatGPT/Claude 웹의 가장 큰 차이는 뭘까?

A 더 똑똑하다
B 내 컴퓨터의 파일을 직접 읽고 쓸 수 있다
C 무료다
D 한국어를 더 잘한다
✅ 정답: B. Claude Code는 터미널에서 실행되기 때문에 파일을 직접 읽고, 쓰고, 명령어를 실행할 수 있어. 채팅 AI는 대화만 가능하지만, Claude Code는 "대화하면서 실행"해.

2 CLAUDE.md — AI에게 주는 내 사용설명서

▼

CLAUDE.md는 프로젝트 폴더에 놓는 마크다운 파일이야. Claude Code가 대화를 시작할 때 자동으로 읽어.

💡 비유: 신입이 들어왔는데 아무것도 안 알려주면 매번 같은 걸 물어봐. 하지만 "내 업무는 이거고, 이런 식으로 일해" 문서를 줘놓으면? 처음부터 맥락을 가지고 일해. CLAUDE.md가 그 문서야.

뭘 쓰면 되나

# About Me

- 이름: [이름]
- 팀: [팀명]
- 역할: [담당 업무 설명]
- 선호 스타일: [예: 간결하게, 표로 정리, 평어 사용 등]

# My Work

- 주요 업무: [매일/매주 하는 일]
- 자주 쓰는 도구: [다우오피스, Teams, Notion 등]
- 주의사항: [예: 고객명은 영문으로, 금액은 만원 단위로 등]

# ABLE Labs Context

- 회사: ABLE Labs (에이블랩스) — 바이오 실험 자동화
- 제품: NOTABLE (액체 핸들링 로봇)
- 구성원 호칭: 에이블러 (ABLER)
- 문화: 평어 문화, 코어타임 자유 출근

예를 들면 이런 걸 넣어

제품 라인업, 자주 쓰는 양식, 팀 컨벤션, 업무 스타일 같은 걸 넣을 수 있어. 에이블러 각자가 자기 역할에 맞는 버전을 만들면 돼. 처음엔 간단하게, 쓰면서 점점 풍부하게.

  1. 1
    Claude Code에서 이렇게 말해봐:
    CLAUDE.md 파일을 만들어줘. 나는 [이름]이고, ABLE Labs [팀]에서 [역할]을 맡고 있어.
  2. 2
    Claude가 초안을 만들어줘. 내용을 확인하고 추가하고 싶은 걸 말해
  3. 3
    완성된 CLAUDE.md를 저장해

💡 처음부터 완벽하지 않아도 돼. 쓰다보면 뭘 넣어야 할지 감이 와. 나중에 계속 업데이트하면 돼.

CLAUDE.md에 넣으면 가장 효과적인 정보는?

A 내 취미와 좋아하는 음식
B 내 역할, 업무, 선호 스타일, 회사 맥락
C Claude Code 사용법
D 비밀번호와 API 키
✅ 정답: B. CLAUDE.md는 AI에게 "내가 누구고 어떤 일을 하는지" 알려주는 파일이야. 업무 맥락이 들어갈수록 AI가 더 정확하게 일해. (D는 절대 넣으면 안 돼! 🚫)

3 기본 도구 4가지

▼

Claude Code에는 핵심 도구 4가지가 있어. 이것만 알면 대부분의 업무를 시킬 수 있어.

도구하는 일비유
Read파일 읽기📄 서류를 꺼내서 읽는 것
Write파일 쓰기✍️ 새 서류를 작성하는 것
Edit파일 수정✏️ 서류의 오타를 수정하는 것
Bash명령어 실행🖨️ 복사기 돌리기, 프린트 등

💡 직접 명령어를 외울 필요 없어. 자연어로 말하면 Claude가 알아서 적절한 도구를 써. "이 파일 읽어줘" → Claude가 Read 도구를 써.

아래 4가지를 순서대로 해봐:

  1. 1
    Read: "이 폴더에 어떤 파일이 있는지 보여줘"
  2. 2
    Write: "오늘 할 일 목록을 todo.md 파일로 만들어줘"
  3. 3
    Edit: "todo.md에서 첫 번째 항목을 완료 표시해줘"
  4. 4
    Bash: "현재 날짜와 시간을 알려줘"

파일의 특정 부분만 수정하고 싶을 때 가장 적절한 도구는?

A Read
B Write
C Edit
D Bash
✅ 정답: C. Edit는 파일의 특정 부분만 수정해. Write는 파일 전체를 새로 쓰는 거고, Edit는 딱 바꿀 부분만 바꿔.

4 실제 업무 1개 해보기

▼

이제 연습이 아니라 진짜 업무를 해봐. 아래에서 지금 당장 해야 하는 업무 1개를 골라서 Claude에게 시켜봐.

업무 예시

누구나직군별
이메일 답장 초안 쓰기생산 일보 양식 초안 (제조)
미팅 회의록 정리실험 프로토콜 요약 (연구)
보고서/문서 초안 작성경비 보고서 정리 (경영지원)
자료 번역 (한↔영)기술 스펙 비교표 (개발)
데이터 정리/요약계약서 조항 요약 (재무)

시키는 법

그냥 자연어로 말하면 돼:

어제 [미팅명] 미팅에서 논의한 내용을 회의록으로 정리해줘.
참석자는 [A, B, C]이고, 주요 안건은 [...]이야.
이 메일에 대한 답장 초안 써줘. 톤은 정중하게.
[메일 내용 붙여넣기]
  1. 1
    지금 해야 하는 업무 1개를 골라
  2. 2
    Claude에게 자연어로 시켜
  3. 3
    결과물을 확인해
  4. 4
    수정할 부분이 있으면 추가 지시해 — "이 부분 수정해줘", "톤을 바꿔줘" 등
  5. 5
    최종 결과물을 파일로 저장해

💡 처음부터 완벽하지 않아도 돼. 대화로 다듬으면 돼. 그게 Claude Code의 진짜 힘이야.
