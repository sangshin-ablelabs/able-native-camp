Level 1 Operate  — 🖥️ 도구 써보기
Level 2 Connect  — 🔌 업무 도구 연결하기
Level 3 Automate — ⚡ 반복 업무 자동화하기
Level 4 Harness  — 🔗 워크플로우 만들기
Level 5 Delegate — 🤖 에이전트에게 위임하기
Level 6 Sustain  — 🌱 성장 환경 만들기

🎯 캠프가 끝나면 이런 게 가능해져

6일 후엔 우리 모두 할 수 있어.

☕

"오늘 뭐 해야 해?"

아침에 Claude한테 물어보면 메일, 일정, 팀즈, 노션을 다 훑어서 오늘의 투두를 정리해줘.

📧

"이 메일 답장 써줘"

맥락을 알고 있으니까 답장 톤, 내용, 참조까지 알아서 맞춰줘. 영어도 한국어도.

📊

"이 데이터 정리해줘"

엑셀, CSV, 텍스트 뭐든 읽고 표로 정리하고, 요약하고, 보고서 초안까지 만들어.

🔄

"매주 금요일에 이거 해줘"

주간 리뷰, 위클리 세팅 같은 반복 업무를 /명령어 하나로 자동 실행해.

🤝

"미팅 준비해줘"

CRM 데이터, 이전 미팅 기록, 메일 히스토리를 종합해서 브리핑 문서를 만들어줘.

🧠

"내가 지난번에 뭐라 했지?"

Memory 시스템이 대화 간 맥락을 기억해. 매번 처음부터 설명할 필요 없어.

📚 커리큘럼

매일 1~2시간. 강의를 듣는 게 아니라, Claude가 1:1로 가르치고 실습을 안내해. 못 따라가면 Claude한테 "다시 설명해줘" 하면 돼.

🎬 Kickoff

Why — 왜 해야 하는가

라이브 시연으로 시작해. 데일리 브리핑 한마디로 뽑기, 해외 미팅 준비 몇 분 만에 끝내기, /명령어로 위클리 리뷰 자동 생성 — 직접 보여줄게. "도구를 배우는 게 아니라, 일하는 방식을 바꾸는 거야."

🎁 산출물: Claude Code 설치 완료 + "이걸 나도 할 수 있겠다" 확신
🎤 라이브 시연 🏔️ 6단계 레벨 소개 💻 설치 확인
1
DAY
🖥️ Level 1: Operate

첫 대화 — "나를 알려주고, 시켜봐"

Claude Code와 첫 대화를 나눠봐. CLAUDE.md를 작성해서 AI에게 "나는 누구고 어떤 일을 하는지" 알려줘. 마치 신입 사원에게 업무 인수인계 문서를 건네는 느낌이야. 기본 도구 4가지(Read, Write, Edit, Bash)를 익히고, 지금 해야 하는 진짜 업무 1개를 시켜봐.

🎁 산출물: CLAUDE.md (나의 AI 사용설명서) + 실전 업무 결과물 1개
💬 첫 대화 📝 CLAUDE.md 작성 🔧 기본 도구 4가지 💼 실전 업무
2
DAY
🔌 Level 2: Connect

연결 — "내 업무 도구를 AI에게 열어줘"

MCP(Model Context Protocol)로 다우오피스, Teams, Notion을 AI에 직접 연결해. 연결하면 "내 메일 확인해줘", "오늘 일정 뭐야", "노션에서 찾아줘"가 진짜로 가능해져. AI가 네 업무 환경을 직접 볼 수 있게 되는 거야.

🎁 산출물: MCP 3개 연동 완료 + "오늘의 업무 요약" 첫 실행
🧩 MCP 개념 📮 다우오피스 (메일/일정) 💬 Teams 📓 Notion 🔀 3개 조합 실습
3
DAY
⚡ Level 3: Automate

자동화 — "반복 업무를 스킬로 만들어"

매주 같은 포맷으로 하는 일이 있지? 그걸 슬래시 커맨드 스킬로 만들어. 한번 만들면 다음부터는 /weekly-report 같은 명령어 하나로 끝. 직군 상관없이 각자 자기 업무에 맞는 스킬을 만들어봐.

🎁 산출물: 나만의 SKILL.md 1개 (동료와 교환 테스트까지)
💡 스킬 개념 📋 SKILL.md 구조 🔍 반복 업무 찾기 🛠️ 스킬 제작 🤝 교환 테스트
4
DAY
🔗 Level 4: Harness

하네스 — "스킬을 엮어 워크플로우를 만들어"

개별 스킬은 부품이야. 하네스가 시스템을 만들어. 스킬 A의 출력이 스킬 B의 입력이 되는 멀티스텝 워크플로우를 구성해봐. 예를 들면: 메일 확인 → 일정 파싱 → 투두 생성이 한번에 돌아가는 거야. 실제 데일리 브리핑 워크플로우를 예시로 보고, 각자 자기 워크플로우를 설계해.

🎁 산출물: 2~3단계 멀티스텝 워크플로우 1개
🧬 하네스 개념 📋 워크플로우 예시 ✏️ 내 워크플로우 설계 🏗️ 빌드 & 테스트
5
DAY
🤖 Level 5: Delegate

위임 — "에이전트에게 맡겨"

여기서부터 차원이 달라져. 스킬은 "이거 해줘". 하네스는 "이 순서로 해줘". 에이전트는 "이 목표를 달성해줘, 방법은 네가 정해." 목표만 주고 AI가 스스로 판단해서 도구를 고르고, 순서를 정하고, 실행까지 하도록 위임하는 법을 익혀보자.

🎁 산출물: 에이전트 지시문 + 자율 실행 결과물
🎯 스킬 vs 에이전트 🔎 위임 가능한 업무 식별 📜 에이전트 인스트럭션 🚀 실전 위임
6
DAY
🌱 Level 6: Sustain

정착 — "복리로 성장하는 환경"

캠프가 끝나도 성장은 계속돼. Memory 시스템으로 AI가 대화 간 맥락을 기억하게 만들고, 아침 루틴·주간 리뷰 같은 개인 루틴을 설정해. 그리고 내가 만든 스킬을 조직과 공유해 — 모든 에이블러가 만든 스킬이 모이면 어마어마한 자산이 돼. 리더보드 최종 결과 발표 + 시상 🏆

🎁 산출물: Memory 정리 + 루틴 설정 + 공용 스킬 등록 + 수료!
🧠 Memory 시스템 ⏰ 개인 루틴 🎪 스킬 공유회 🎓 졸업식

🏔️ 레벨별 역량 매트릭스

각 레벨을 달성하면 할 수 있는 것들이야. 하나씩 올라가면 돼.

Level 이름 할 수 있는 것 핵심 산출물
Level 1 Operate Claude Code로 대화, 파일 읽기/쓰기, 간단한 업무 요청 CLAUDE.md
Level 2 Connect MCP로 다우/팀즈/노션 연결, 실시간 데이터 조회 MCP 3개 연동
Level 3 Automate 반복 업무를 스킬로 만들어 슬래시 커맨드로 실행 SKILL.md 1개

Level 5 Delegate 목표만 주고 AI가 판단해서 실행하도록 위임 에이전트 지시문
Level 6 Sustain Memory/루틴으로 지속적 성장, 조직 스킬 기여 공유 스킬 등록

🤔 잠깐 — 터미널이 뭐야?

터미널은 컴퓨터에게 글자로 명령을 내리는 창이야. 마우스 대신 키보드로 "이거 해" 하면 컴퓨터가 실행해.
Claude Code는 이 터미널 안에서 돌아가. 아래처럼 생겼어:

Terminal
~ $ claude
╭──────────────────────────────╮
│ Welcome to Claude Code │
╰──────────────────────────────╯
> 무엇을 도와줄까?_

🍎 Mac

⌘ Command + Space → "터미널" 검색

또는 Finder → 응용 프로그램 → 유틸리티 → 터미널

🪟 Windows

Win + X → "Windows PowerShell" 선택

또는 시작 버튼 → "PowerShell" 검색. (cmd가 아닌 PowerShell을 사용해야 해)

💡 터미널에 붙여넣기: Mac은 ⌘+V / Windows는 마우스 우클릭 또는 Ctrl+Shift+V

1

터미널을 열고, 아래를 복사 → 붙여넣기 → Enter

🍎 Mac / Linux:

curl -fsSL https://claude.ai/install.sh | bash

🪟 Windows PowerShell:

irm https://claude.ai/install.ps1 | iex
2

캠프 프로젝트를 내 컴퓨터로 가져와. 복사 → 붙여넣기 → Enter

git clone https://github.com/sangshin-ablelabs/able-native-camp.git

💡 git이 없다면? Mac: xcode-select --install / Windows: git-scm.com

3

프로젝트 폴더로 들어가서 Claude Code 시작! 복사 → 붙여넣기 → Enter

cd able-native-camp && claude
🔑 처음 실행하면 로그인이 필요해:
• 브라우저가 자동으로 열리면 Anthropic 계정으로 로그인
• 계정이 없으면 claude.ai 에서 먼저 가입
• Claude Pro / Max / Teams 구독이 필요해 (회사에서 제공하는 경우 회사 안내를 따르면 돼)
4

Claude가 뜨면, Day 0 스킬을 실행해봐! 아래를 입력 → Enter

/day0-why

Day 0 → Day 1 → Day 2 … 순서대로 진행하면 돼. 각 Day는 /day1-operate 같은 명령어로 시작해.
