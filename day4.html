Day 4 — 하네스

"스킬을 연결해서 워크플로우를 만들어."
단일 스킬을 엮어서 여러 단계를 자동으로 실행하는 날이야.

하네스란 브리핑 예시 설계 실전 구축 마무리

1 🔗 하네스란 — 스킬을 연결하는 기술

▼

Harness(하네스)는 자동차의 와이어링 하네스처럼 스킬을 연결하는 거야. 자동차에서 수십 개의 전선을 하나로 묶어서 전기 신호를 전달하듯, AI 스킬도 연결해서 결과물을 전달할 수 있어.

핵심은 이거야: Skill A의 결과물이 Skill B의 입력이 되는 것.

단일 스킬은 "한 가지"만 해. 하네스는 "여러 가지를 순서대로" 해.

단일 스킬 vs 하네스

단일 스킬하네스 (워크플로우)
/주간보고 → 보고서 1개 생성캘린더→메일→Teams→요약→노션 전체 자동
한 번에 한 가지 작업여러 단계를 순서대로 자동 실행
결과물이 거기서 끝결과물이 다음 스킬의 입력이 됨
레고 블록 1개레고로 만든 완성품

💡 스킬이 레고 블록이라면, 하네스는 레고로 만든 완성품이야.

  1. 1
    지금까지 만든 스킬 목록을 확인해봐. ls ~/.claude/commands/ 또는 프로젝트의 .claude/commands/ 확인
  2. 2
    "이 스킬들을 순서대로 연결하면?" 생각해봐. 어떤 스킬의 결과물이 다른 스킬의 입력이 될 수 있을까?
  3. 3
    연결 가능한 조합을 찾아봐. 예: 메일요약 → 보고서작성, 일정확인 → 미팅준비

하네스가 단일 스킬과 다른 점은?

A 더 빠르다
B 스킬 A의 결과물이 스킬 B의 입력이 된다
C MCP가 필요없다
D 코딩이 필요하다
✅ 정답: B. 하네스의 핵심은 스킬 간 연결이야. Skill A의 결과물이 Skill B의 입력이 되면서, 여러 단계가 자동으로 이어져. 단순히 빠른 게 아니라 "연결"이 핵심이야.

2 📋 워크플로우 예시 — 데일리 브리핑

▼

실제로 쓸 수 있는 워크플로우 예시를 보여줄게. 데일리 브리핑 — 한마디면 5단계가 자동으로 돌아가.

5단계 연결 흐름

Step 1: 📅 캘린더 확인 (오늘 일정)
     ↓ 일정 목록
Step 2: 📬 메일 확인 (읽지 않은 + 중요)
     ↓ 메일 요약
Step 3: 💬 Teams 확인 (멘션 + 미회신)
     ↓ 메시지 정리
Step 4: 📊 종합 요약 (우선순위 정리)
     ↓ 요약 결과
Step 5: 📓 노션 투두 생성 (체크리스트)

시간 비교

방법소요 시간
수작업 (각각 열어서 확인)약 20분
하네스 (한마디)약 3분

💡 한 마디만 하면 5단계가 자동 실행돼. 이게 하네스의 힘이야.

  1. 1
    Claude에게 "오늘 업무 브리핑해줘. 캘린더→메일→Teams→종합요약→투두리스트 순서로" 라고 시켜봐
  2. 2
    각 단계가 연결되는 과정을 관찰해. Step 1의 결과가 Step 4의 종합 요약에 포함되는지 확인해
  3. 3
    결과물이 어떻게 이어지는지 확인해. 캘린더 일정이 투두 체크리스트에 반영됐는지 봐

💡 처음엔 각 단계별로 결과를 보여달라고 해. "각 단계 결과를 따로 보여주고, 마지막에 종합해줘"

3 🗺️ 내 워크플로우 설계 — 연결점 찾기

▼

워크플로우를 설계하려면 3단계를 거쳐야 해:

Step 1. 반복 패턴 찾기

매주, 매월 같은 순서로 하는 일이 뭐야? 그게 워크플로우 후보야.

Step 2. 단계 분리

그 일을 개별 작업으로 쪼개. "주간보고 쓴다" → 메일 확인 + 일정 정리 + 팀즈 확인 + 보고서 작성 + 저장

Step 3. 연결점 찾기

어떤 결과물이 다음 단계의 입력이 되는지 파악해. 이게 하네스의 핵심이야.

워크플로우 예시

워크플로우단계 흐름
주간보고메일요약 → 일정정리 → 팀즈확인 → 보고서작성 → 노션저장
생산일지설비데이터 → 생산량집계 → 불량분석 → 일지작성
고객문의문의접수 → CRM확인 → 답변초안 → 팀리뷰 → 발송

⚠️ 처음부터 5단계 이상 만들지 마. 2-3단계부터 시작해서 점점 늘려가.

  1. 1
    내 반복 업무 중 "여러 단계를 순서대로" 하는 것 1개를 선택해
  2. 2
    그 업무를 단계별로 분리해봐. "이 일을 하려면 먼저 뭘 하고, 다음에 뭘 하지?"
  3. 3
    각 단계의 입력(뭐가 필요해)과 출력(뭐가 나와)을 정리해
  4. 4
    Claude에게 "이 워크플로우를 스킬로 만들어줘" 라고 말해봐

워크플로우 설계 시 가장 먼저 해야 할 것은?

A 코드 작성
B 반복 패턴과 연결점 찾기
C MCP 추가 설치
D 새 프로젝트 생성
✅ 정답: B. 코드나 도구보다 먼저 해야 할 건 "어떤 일이 반복되는지" 패턴을 찾고, "어떤 결과물이 다음 단계로 이어지는지" 연결점을 파악하는 거야. 설계가 먼저, 구현은 나중이야.

4 🛠️ 워크플로우 만들기 — 실전 구축

▼

이제 실제로 워크플로우 스킬을 만들어볼 거야. 핵심 원칙 3가지:

  • 2-3단계로 시작해. 완벽하게 5단계부터 시작하면 디버깅이 어려워.
  • 각 단계의 입력/출력을 명확히 해. "뭘 받아서 뭘 내보내는지"가 핵심이야.
  • 필요하면 중간에 사람 확인 단계를 넣어. 안전장치야.

워크플로우 SKILL.md 예시

# /daily-briefing

## 목적
매일 아침 업무 현황을 종합 브리핑한다.

## 워크플로우

### Step 1: 캘린더 확인
- 입력: 오늘 날짜
- 실행: 다우오피스 캘린더에서 오늘 일정 조회
- 출력: 일정 목록 → 다음 단계로 전달

### Step 2: 메일 확인
- 입력: Step 1의 일정 목록 (관련 메일 필터용)
- 실행: 읽지 않은 메일 + 중요 메일 조회
- 출력: 메일 요약 → 다음 단계로 전달

### Step 3: 종합 요약
- 입력: Step 1 일정 + Step 2 메일
- 실행: 우선순위 정리, 투두 리스트 생성
- 출력: 오늘의 브리핑 결과

각 단계 마지막에 "→ 다음 단계로 전달" 형식을 써서 흐름을 명확히 해.

💡 완벽하지 않아도 돼. 돌아가게 만들고, 쓰면서 개선하는 거야.

  1. 1
    Block 2에서 설계한 워크플로우를 SKILL.md 형식으로 작성해봐
  2. 2
    Claude에게 "이 워크플로우 스킬 만들어줘" 라고 시켜봐
  3. 3
    만들어진 스킬을 실행해봐. /워크플로우이름
  4. 4
    안 되는 부분이 있으면 수정해. "Step 2에서 결과가 안 넘어가" 같은 피드백을 줘
  5. 5
    2-3회 반복해서 다듬어. 처음부터 완벽할 필요 없어

💡 "이 스킬 실행하고, 각 단계별 결과를 보여줘" 라고 하면 디버깅이 쉬워져.

워크플로우 스킬 작성 시 핵심은?

A 최대한 많은 단계
B 각 단계의 입력/출력을 명확히
C 영어로 작성
D MCP 10개 이상 연결
✅ 정답: B. 단계가 많거나 MCP가 많다고 좋은 게 아니야. 각 단계에서 "뭘 받고 뭘 내보내는지"가 명확해야 스킬이 제대로 연결돼. 입력/출력이 불분명하면 연결이 끊겨.
