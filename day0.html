0 AI Native란 무엇인가? — 왜 지금 AI Native인가

▼

"AI 잘 쓰는 사람"과 "AI Native"는 다르다. AI Native는 일하는 방식 자체가 AI와 함께 설계된 것을 말해.

기존 방식 vs AI Native 방식

구분기존 방식AI Native 방식
문서 작성빈 화면에서 시작AI가 초안 → 사람이 다듬기
데이터 분석엑셀 수식 직접 작성"이 데이터 트렌드 분석해줘"
이메일 답장처음부터 타이핑AI 초안 → 톤/내용 조정
코드 작성한 줄씩 직접 코딩자연어로 설명 → AI가 구현
정보 검색구글링 → 여러 페이지 읽기"~에 대해 정리해줘"

핵심: AI Native는 "AI를 가끔 쓰는 것"이 아니라, "AI 없이는 일하지 않는 것"이야. 마치 이메일 없이 업무가 안 되듯이.

왜 지금인가?

  • 2024~2025: AI가 "대화"에서 "실행"으로 진화했어. 파일을 읽고, 쓰고, 명령어를 실행할 수 있게 됐어.
  • Claude Code: 터미널에서 AI가 직접 내 컴퓨터를 조작해. 채팅 AI와는 차원이 달라.
  • MCP (Model Context Protocol): AI가 이메일, 캘린더, 노션, Teams 등에 직접 연결돼. 복사-붙여넣기가 사라져.

ABLE Labs는 왜 AI Native를 해야 하나?

우리는 실험 자동화 회사야. 고객에게 "자동화하세요"라고 말하면서 정작 우리 업무는 수동이면 설득력이 없어. AI Native는 우리의 정체성에 맞는 일하는 방식이야.

  • 30명 규모의 스타트업 — 한 사람이 여러 역할을 해. AI가 배수기 역할을 해줘야 해.
  • 글로벌 비즈니스 — 영어/일본어 커뮤니케이션을 AI가 자연스럽게 지원해.
  • 기술 회사 — 코드, 문서, 데이터를 다루는 업무가 많아. AI가 가장 잘하는 영역이야.

"AI Native"의 핵심 정의로 가장 적절한 것은?

A ChatGPT를 매일 쓰는 것
B AI와 함께 일하는 방식 자체를 설계하는 것
C 코딩을 AI로 대체하는 것
D 모든 업무를 AI에게 맡기는 것
✅ 정답: B. AI Native는 특정 도구를 쓰는 게 아니라, 업무 프로세스 자체를 AI와 협업하는 방식으로 재설계하는 것이야. "가끔 쓰는 도구"가 아니라 "기본 인프라"로 자리잡는 것.

1 데모 — Claude가 실제로 뭘 할 수 있는지

▼

백문이 불여일견. Claude Code가 실제로 어떤 일을 하는지 시나리오로 살펴보자.

시나리오 1: 이메일 초안 작성

# 사람이 하는 말:
"거래처 A사에 납품 일정 변경 안내 메일 초안 써줘.
기존: 3/25 → 변경: 4/1. 사유는 원자재 수급 지연."

# Claude가 하는 일:
→ 메일 제목 + 본문 초안 작성 (HTML 파일로 저장)
→ 톤 조정 요청하면 즉시 수정
→ "영문 버전도 만들어줘" → 바로 번역

시나리오 2: 데이터 정리

# 사람이 하는 말:
"이 CSV 파일 읽어서 월별 매출 요약표 만들어줘"

# Claude가 하는 일:
→ CSV 파일 직접 읽기 (Read 도구)
→ 데이터 분석 및 요약
→ 마크다운 표 또는 새 파일로 결과 저장 (Write 도구)

시나리오 3: 회의록 정리

# 사람이 하는 말:
"아까 미팅 내용 정리해줘. 참석자는 A, B, C.
논의 사항: 1) Q2 목표, 2) 신규 채용, 3) 제품 출시 일정"

# Claude가 하는 일:
→ 구조화된 회의록 작성
→ 액션 아이템 정리
→ 담당자별 할 일 리스트 생성

시나리오 4: MCP 연결 후

# MCP로 다우오피스, Teams, Notion이 연결되면:

"오늘 메일 중 답장 안 한 거 있어?"
→ 다우오피스 메일함 직접 확인 → 미회신 메일 리스트 제공

"이번 주 일정 정리해줘"
→ 캘린더 직접 조회 → 일정표 생성

"Teams에서 나한테 멘션된 거 확인해줘"
→ Teams 멘션 조회 → 요약 제공

핵심: Claude Code는 "답변"만 하는 게 아니라, 내 컴퓨터에서 직접 "실행"한다. 파일을 읽고, 쓰고, 외부 서비스에 접근해.

Claude Code가 일반 ChatGPT/웹 Claude와 가장 다른 점은?

A 더 긴 답변을 해준다
B 내 컴퓨터에서 파일을 읽고 쓰고 명령어를 실행할 수 있다
C 이미지를 생성할 수 있다
D 인터넷 검색을 할 수 있다
✅ 정답: B. Claude Code는 터미널에서 실행되기 때문에 로컬 파일 시스템에 직접 접근할 수 있어. 채팅 AI는 대화만 하지만, Claude Code는 실제로 파일을 읽고, 수정하고, 새로 만들어.

2 네이티브 레벨 — Level 1 ~ Level 6

▼

ABLE Native Camp은 7일간 6단계 레벨을 올라가는 과정이야. Day 0은 킥오프, Day 1부터 매일 한 레벨씩. 각 레벨이 뭘 의미하는지 알아보자.

Lv.1

Operate — 조작

Claude Code를 직접 실행하고, 기본 대화를 나눌 수 있다. CLAUDE.md를 만들고, 기본 도구(Read/Write/Edit/Bash)를 사용할 수 있다.

Lv.2

Connect — 연결

MCP로 외부 도구(다우오피스, Teams, Notion)를 연결할 수 있다. AI가 내 업무 환경에 직접 접근해.

Lv.3

Automate — 자동화

반복 업무를 스킬(슬래시 커맨드)로 만들어서 한 번에 실행할 수 있다. 매일 하는 일을 자동화해.

Lv.4

Harness — 연결

스킬을 엮어 멀티스텝 워크플로우를 만들 수 있다. 스킬 A의 출력이 스킬 B의 입력이 되는 체인을 구성해.

Lv.5

Delegate — 위임

목표만 주고 AI가 스스로 판단해서 도구를 고르고, 순서를 정하고, 실행까지. 에이전트에게 업무를 통째로 맡길 수 있다.

Lv.6

Sustain — 정착

Memory 시스템으로 대화 간 맥락을 기억하고, 개인 루틴을 설정하고, 스킬을 조직과 공유해. 복리로 성장하는 환경을 만든다.

3 설치 & 첫 실행 — Claude Code 시작하기

▼

Step 0: 터미널 열기 (처음인 사람은 여기부터!)

터미널은 텍스트로 컴퓨터에 명령을 내리는 창이야. 처음이라 낯설 수 있지만 금방 익숙해져.

운영체제여는 방법
🍎 MacCmd + Space → "터미널" 검색 → Enter
또는 Finder → 응용 프로그램 → 유틸리티 → 터미널
🪟 Windows시작 버튼 옆 검색창에 "PowerShell" 검색 → Windows PowerShell 클릭
또는 Win + R → powershell 입력 → Enter

💡 VS Code를 이미 쓰고 있다면? Ctrl + ` (백틱)으로 터미널을 바로 열 수 있어. 이게 가장 편해!

Step 1: Node.js 설치

Claude Code는 Node.js 위에서 돌아가. 먼저 설치되어 있는지 확인하자.

# 터미널에서 실행
node --version

# v18 이상이면 OK

버전이 안 나오거나 v18 미만이면 설치가 필요해:

👉 Node.js 다운로드 페이지 (https://nodejs.org/ko/download)

  • Mac: macOS Installer (.pkg) 다운로드 → 더블클릭 → 설치 완료
  • Windows: Windows Installer (.msi) 다운로드 → 더블클릭 → "Next" 계속 클릭 → 설치 완료
  • 설치 후 터미널을 껐다가 다시 열고 node --version으로 확인

Step 1.5: Git 설치

캠프 자료를 다운받으려면 Git이 필요해.

# 터미널에서 실행
git --version

# 버전이 나오면 OK

안 나오면 설치가 필요해:

운영체제설치 방법
🪟 WindowsPowerShell에서 winget install Git.Git 실행 (가장 간편!)
또는 git-scm.com/downloads에서 다운로드
🍎 Mac터미널에서 xcode-select --install 실행
또는 git-scm.com/downloads에서 다운로드

설치 후 터미널을 껐다가 다시 열고 git --version으로 확인해.

Step 2: Claude Code 설치

npm install -g @anthropic-ai/claude-code

이 한 줄이면 설치 완료.

⚠️ Windows 사용자: 반드시 PowerShell을 사용해. cmd(명령 프롬프트)는 일부 명령어가 달라서 문제가 생길 수 있어. 시작 메뉴에서 "PowerShell"을 검색해서 열자.

Step 3: 캠프 자료 다운로드

캠프 실습 자료를 내 컴퓨터에 받아야 해.

# PowerShell 또는 터미널에서 실행
git clone https://github.com/sangshin-ablelabs/able-native-camp.git

실행하면 현재 위치에 able-native-camp 폴더가 생겨.

Step 4: 캠프 폴더에서 터미널 열기

⚠️ 중요: cd able-native-camp 명령어가 익숙하지 않으면, 아래 방법을 추천해:

운영체제폴더에서 바로 터미널 여는 법
🪟 Windows파일 탐색기에서 able-native-camp 폴더를 찾아 → 폴더 안에서 빈 공간 우클릭 → "터미널에서 열기" 또는 "여기에서 PowerShell 창 열기"
🍎 MacFinder에서 able-native-camp 폴더를 찾아 → 폴더를 우클릭 → "폴더에서 새로운 터미널 열기"
또는 터미널에서 cd ~/able-native-camp
💻 VS CodeVS Code에서 파일 → 폴더 열기 → able-native-camp 폴더 선택 → Ctrl + ` 로 터미널 열기 (가장 추천!)

💡 폴더가 어디 있는지 모르겠다면? git clone을 실행한 위치에 생겨. Windows에서는 보통 C:\Users\[내이름] 폴더, Mac에서는 /Users/[내이름] 폴더 안에 있어.

Step 5: 계정 연결 + 첫 실행

# able-native-camp 폴더에서 터미널이 열려 있는 상태에서
claude

# 처음 실행하면 Anthropic 계정 로그인 안내가 나와.
# 회사 계정으로 로그인하면 돼.

Step 6: 첫 대화

# Claude Code가 실행되면
안녕! 나는 [이름]이야. 잘 부탁해.

축하해! 이게 AI Native의 첫 걸음이야. 내일(Day 1)부터 본격적으로 시작하자.
