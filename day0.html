0 AI Native란 무엇인가? — 왜 지금 AI Native인가

▼

"AI 잘 쓰는 사람"과 "AI Native"는 다르다. AI Native는 일하는 방식 자체가 AI와 함께 설계된 것을 말해.

기존 방식 vs AI Native 방식

구분기존 방식AI Native 방식
문서 작성빈 화면에서 시작AI가 초안 → 사람이 다듬기
데이터 분석엑셀 수식 직접 작성"이 데이터 트렌드 분석해줘"
이메일 답장처음부터 타이핑AI 초안 → 톤/내용 조정
코드 작성한 줄씩 직접 코딩자연어로 설명 → AI가 구현
정보 검색구글링 → 여러 페이지 읽기"~에 대해 정리해줘"

핵심: AI Native는 "AI를 가끔 쓰는 것"이 아니라, "AI 없이는 일하지 않는 것"이야. 마치 이메일 없이 업무가 안 되듯이.

왜 지금인가?

  • 2024~2025: AI가 "대화"에서 "실행"으로 진화했어. 파일을 읽고, 쓰고, 명령어를 실행할 수 있게 됐어.
  • Claude Code: 터미널에서 AI가 직접 내 컴퓨터를 조작해. 채팅 AI와는 차원이 달라.
  • MCP (Model Context Protocol): AI가 이메일, 캘린더, 노션, Teams 등에 직접 연결돼. 복사-붙여넣기가 사라져.

ABLE Labs는 왜 AI Native를 해야 하나?

우리는 실험 자동화 회사야. 고객에게 "자동화하세요"라고 말하면서 정작 우리 업무는 수동이면 설득력이 없어. AI Native는 우리의 정체성에 맞는 일하는 방식이야.

  • 30명 규모의 스타트업 — 한 사람이 여러 역할을 해. AI가 배수기 역할을 해줘야 해.
  • 글로벌 비즈니스 — 영어/일본어 커뮤니케이션을 AI가 자연스럽게 지원해.
  • 기술 회사 — 코드, 문서, 데이터를 다루는 업무가 많아. AI가 가장 잘하는 영역이야.

"AI Native"의 핵심 정의로 가장 적절한 것은?

A ChatGPT를 매일 쓰는 것
B AI와 함께 일하는 방식 자체를 설계하는 것
C 코딩을 AI로 대체하는 것
D 모든 업무를 AI에게 맡기는 것
✅ 정답: B. AI Native는 특정 도구를 쓰는 게 아니라, 업무 프로세스 자체를 AI와 협업하는 방식으로 재설계하는 것이야. "가끔 쓰는 도구"가 아니라 "기본 인프라"로 자리잡는 것.

1 데모 — Claude가 실제로 뭘 할 수 있는지

▼

백문이 불여일견. Claude Code가 실제로 어떤 일을 하는지 시나리오로 살펴보자.

시나리오 1: 이메일 초안 작성

# 사람이 하는 말:
"거래처 A사에 납품 일정 변경 안내 메일 초안 써줘.
기존: 3/25 → 변경: 4/1. 사유는 원자재 수급 지연."

# Claude가 하는 일:
→ 메일 제목 + 본문 초안 작성 (HTML 파일로 저장)
→ 톤 조정 요청하면 즉시 수정
→ "영문 버전도 만들어줘" → 바로 번역

시나리오 2: 데이터 정리

# 사람이 하는 말:
"이 CSV 파일 읽어서 월별 매출 요약표 만들어줘"

# Claude가 하는 일:
→ CSV 파일 직접 읽기 (Read 도구)
→ 데이터 분석 및 요약
→ 마크다운 표 또는 새 파일로 결과 저장 (Write 도구)

시나리오 3: 회의록 정리

# 사람이 하는 말:
"아까 미팅 내용 정리해줘. 참석자는 A, B, C.
논의 사항: 1) Q2 목표, 2) 신규 채용, 3) 제품 출시 일정"

# Claude가 하는 일:
→ 구조화된 회의록 작성
→ 액션 아이템 정리
→ 담당자별 할 일 리스트 생성

시나리오 4: MCP 연결 후

# MCP로 다우오피스, Teams, Notion이 연결되면:

"오늘 메일 중 답장 안 한 거 있어?"
→ 다우오피스 메일함 직접 확인 → 미회신 메일 리스트 제공

"이번 주 일정 정리해줘"
→ 캘린더 직접 조회 → 일정표 생성

"Teams에서 나한테 멘션된 거 확인해줘"
→ Teams 멘션 조회 → 요약 제공

핵심: Claude Code는 "답변"만 하는 게 아니라, 내 컴퓨터에서 직접 "실행"한다. 파일을 읽고, 쓰고, 외부 서비스에 접근해.

Claude Code가 일반 ChatGPT/웹 Claude와 가장 다른 점은?

A 더 긴 답변을 해준다
B 내 컴퓨터에서 파일을 읽고 쓰고 명령어를 실행할 수 있다
C 이미지를 생성할 수 있다
D 인터넷 검색을 할 수 있다
✅ 정답: B. Claude Code는 터미널에서 실행되기 때문에 로컬 파일 시스템에 직접 접근할 수 있어. 채팅 AI는 대화만 하지만, Claude Code는 실제로 파일을 읽고, 수정하고, 새로 만들어.

2 네이티브 레벨 — Lv.0 ~ Lv.4

▼

ABLE Native Camp은 6일간 레벨 0에서 레벨 4까지 올라가는 과정이야. 각 레벨이 뭘 의미하는지 알아보자.

Lv.0

Aware — 인식

AI가 뭘 할 수 있는지 안다. "이런 게 가능하구나"를 이해하는 단계. 오늘(Day 0)이 여기야.

Lv.1

Operate — 조작

Claude Code를 직접 실행하고, 기본 대화를 나눌 수 있다. CLAUDE.md를 만들고, 기본 도구(Read/Write/Edit/Bash)를 사용할 수 있다.

Lv.2

Connect — 연결

MCP로 외부 도구(다우오피스, Teams, Notion)를 연결할 수 있다. AI가 내 업무 환경에 직접 접근해.

Lv.3

Automate — 자동화

반복 업무를 스킬(슬래시 커맨드)로 만들어서 한 번에 실행할 수 있다. 매일 하는 일을 자동화해.

Lv.4

Architect — 설계

팀/조직 단위의 AI 워크플로우를 설계할 수 있다. 다른 에이블러에게 전파하고, 조직의 일하는 방식을 바꿀 수 있다.

3 설치 & 첫 실행 — Claude Code 시작하기

▼

Step 1: Node.js 설치 확인

Claude Code는 Node.js 위에서 돌아가. 먼저 설치되어 있는지 확인하자.

# 터미널에서 실행
node --version

# v18 이상이면 OK. 없으면 https://nodejs.org 에서 설치

Step 2: Claude Code 설치

npm install -g @anthropic-ai/claude-code

이 한 줄이면 설치 완료.

Step 3: 계정 연결

# 아무 폴더에서
claude

# 처음 실행하면 Anthropic 계정 로그인 안내가 나와.
# 회사 계정으로 로그인하면 돼.

Step 4: VS Code에서 터미널 열기

VS Code를 쓰고 있다면, 내장 터미널이 가장 편해:

  • Mac: Ctrl + ` (백틱)
  • 또는 메뉴 → 터미널 → 새 터미널

VS Code 터미널에서 claude를 실행하면, 현재 열려 있는 프로젝트 폴더가 자동으로 작업 디렉토리가 돼. 파일을 바로 읽고 수정할 수 있어서 가장 실용적인 조합이야.

Step 5: 첫 대화

# Claude Code가 실행되면
안녕! 나는 [이름]이야. 잘 부탁해.

축하해! 이게 AI Native의 첫 걸음이야. 내일(Day 1)부터 본격적으로 시작하자.

  1. 1
    터미널을 열어 (VS Code: Ctrl + `)
  2. 2
    node --version 으로 Node.js 확인. 없으면 nodejs.org에서 설치
  3. 3
    npm install -g @anthropic-ai/claude-code 로 Claude Code 설치
  4. 4
    claude 를 입력해서 실행. 계정 연결이 필요하면 안내에 따라 로그인
  5. 5
    자기소개를 하고 간단한 질문 2~3개 해봐

💡 설치가 안 되면 당황하지 말고, Teams #able-native-camp 채널에 물어봐. 누군가 바로 도와줄 거야.
