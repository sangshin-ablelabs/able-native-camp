1 스킬 vs 에이전트 — 실행과 판단의 차이

▼

AI 활용에는 4단계 계층이 있어. 각 단계마다 AI에게 주는 자율성이 달라져.

4단계 계층

단계지시 방식예시
Skill"이거 해"메일 요약해, 번역해
Harness"순서대로 해"메일 확인 → 요약 → 보고서
Agent"이 목표 달성해, 방법은 네가 판단"놓친 거 없는지 확인해줘
Sub-agent에이전트가 에이전트에게 위임메일 에이전트 + 일정 에이전트 협업

핵심 차이: 스킬/하네스는 사람이 방법을 정해주지만, 에이전트는 AI가 스스로 방법을 판단해.

스킬 vs 에이전트 비교

스킬에이전트
실행 방식지시대로 실행목표만 주면 방법 자체 판단
순서정해진 순서상황 따라 다르게 행동
판단사람이 판단AI가 판단
유연성고정적적응적

스킬은 "레시피대로 요리해", 에이전트는 "맛있는 저녁 만들어" — 재료 보고 알아서 판단하는 거야.

  1. 1
    지금까지 만든 스킬/워크플로우를 떠올려봐
  2. 2
    "방법을 내가 정해줬는데, AI가 알아서 해도 될까?" 생각해봐
  3. 3
    그 차이를 느껴봐 — 지시 vs 위임의 경계가 어디인지

에이전트가 스킬/하네스와 다른 핵심 차이는?

A 더 빠르다
B AI가 방법을 스스로 판단한다
C MCP가 더 많이 필요하다
D 코딩이 필요하다
✅ 정답: B. 에이전트의 핵심은 AI가 목표만 받고 방법을 스스로 판단한다는 거야. 스킬은 정해진 대로 실행하지만, 에이전트는 상황을 보고 어디를 확인할지, 뭘 먼저 할지 스스로 결정해.

2 🤖 에이전트 데모 — 이번 주 놓친 거 없어?

▼

실제 데모를 보자. "이번 주 놓친 거 없는지 확인해줘" — 이 한마디면 에이전트가 자율적으로 움직여.

에이전트의 자율 판단 과정

순서에이전트 행동판단 내용
1📅 캘린더 확인일정 놓친 건 없나?
2📬 메일 확인미회신 메일 있나?
3💬 Teams 확인미응답 멘션 있나?
4🎯 우선순위 판단긴급 / 보통 / 참고 분류
5📝 리포트 작성종합 정리

핵심: 사람은 "확인해줘"만 말했는데, AI가 어디서 뭘 볼지, 뭐가 긴급한지 스스로 판단했어.

에이전트가 잘 작동하려면: CLAUDE.md(뇌) + MCP(손) + 명확한 목표(지시) — 3가지가 갖춰져야 해.

  1. 1
    Claude에게 "이번 주 놓친 게 없는지 확인해줘"라고 말해봐
  2. 2
    AI가 어떤 순서로, 어디를 확인하는지 관찰해
  3. 3
    판단 기준이 어떻게 작동하는지 봐 — 긴급/보통/참고를 어떻게 나눴어?

에이전트의 행동 패턴을 관찰하는 게 이 실습의 핵심이야. 뭘 먼저 했고, 왜 그랬는지 생각해봐.

3 위임 가능한 업무 판별 — 체크리스트

▼

모든 업무를 에이전트에게 맡길 수 있는 건 아니야. 2개 이상 해당하면 위임 가능해:

위임 가능 체크리스트

  • ✅ 정답이 여러 개인 업무
  • ✅ 정보 수집 + 분석 + 판단이 섞인 업무
  • ✅ 매번 다르게 진행되는 업무
  • ✅ 목표는 명확하지만 과정은 유연한 업무
  • ✅ 실수해도 되돌릴 수 있는 업무

절대 위임하면 안 되는 것

  • ❌ 최종 결정 (승인, 계약, 채용)
  • ❌ 민감 데이터 처리
  • ❌ 되돌릴 수 없는 행동 (전송, 삭제)
  • ❌ 감정 판단이 필요한 것

핵심 원칙: "에이전트가 초안, 사람이 결정." 이 경계를 지키면 안전하게 위임할 수 있어.

  1. 1
    내 업무 중 위임 가능한 것 3개를 적어봐
  2. 2
    위 체크리스트로 검증해 — 2개 이상 해당하는지 확인
  3. 3
    1개 선택해서 Claude에게 "이 업무 네가 판단해서 해봐"라고 시켜봐

AI에게 위임하면 안 되는 것은?

A 데이터 수집과 분석
B 최종 승인 결정
C 초안 작성
D 정보 요약
✅ 정답: B. 최종 승인, 계약, 채용 같은 결정은 반드시 사람이 해야 해. AI는 초안과 분석까지만 — 최종 결정은 항상 사람의 몫이야.

4 📝 에이전트 지시문 작성 — CLAUDE.md에 판단 기준 넣기

▼

에이전트가 제대로 일하려면 CLAUDE.md에 판단 기준을 넣어야 해. 4가지 필수 요소가 있어:

에이전트 지시문 4요소

#요소설명예시
1목표뭘 완수할지이번 주 놓친 업무 리포트 작성
2사용 가능한 도구MCP, 스킬, 파일 접근다우오피스, Teams, Notion
3판단 기준우선순위 규칙, 긴급 기준, 예외 처리24시간 미회신 = 긴급
4제약조건못 하는 것, 확인 필요한 것, 형식 제한메시지 전송 전 반드시 확인

CLAUDE.md 에이전트 섹션 템플릿

## 주간 리포트 에이전트

### 목표
매주 금요일, 이번 주 놓친 업무가 없는지 확인하고 리포트 작성

### 사용 도구
- 다우오피스: 캘린더, 메일
- Teams: 멘션, 채널 메시지
- Notion: 태스크 DB

### 판단 기준
- 24시간 이상 미회신 메일 → 긴급
- 멘션 미응답 → 긴급
- 일정 충돌 → 보통
- 참고용 정보 → 참고

### 제약조건
- 메시지 전송 전 반드시 본인 확인
- 민감 정보 포함 시 알림만
- 결과물은 노션 페이지로 정리

CLAUDE.md는 에이전트의 '뇌', 지시문은 에이전트의 '업무 매뉴얼'. 둘 다 있어야 제대로 작동해.

  1. 1
    Block 2에서 선택한 위임 업무로 지시문을 작성해봐
  2. 2
    4가지 요소(목표, 도구, 판단 기준, 제약조건)를 모두 포함시켜
  3. 3
    CLAUDE.md에 추가해
  4. 4
    실행해보고 AI의 판단 과정을 관찰해
  5. 5
    부족한 판단 기준이 있으면 보충해 — 반복할수록 정교해져

처음부터 완벽할 필요 없어. 실행 → 관찰 → 보충을 반복하면 에이전트가 점점 똑똑해져.

에이전트 지시문에 반드시 포함해야 하는 것은?

A 코드 예시
B 목표, 도구, 판단기준, 제약조건
C 실행 시간
D 에이전트 이름
✅ 정답: B. 에이전트 지시문에는 목표(뭘 할지), 도구(뭘 쓸 수 있는지), 판단 기준(어떻게 판단할지), 제약조건(못 하는 것)이 반드시 포함돼야 해.
