1 MCP란 — AI에게 손을 달아주는 것

▼

MCP는 Model Context Protocol의 약자야. AI에게 "손"을 달아주는 규격이라고 생각하면 돼.

💡 비유: 스마트폰에 앱을 설치하듯, Claude Code에 MCP를 연결하면 새로운 능력이 생겨.

MCP 연결 전 vs 후

연결 전 ❌연결 후 ✅
"메일 확인해줘" → 못 해"메일 확인해줘" → 바로 조회
"오늘 일정 뭐야" → 모름"오늘 일정 뭐야" → 캘린더 확인
"팀즈에서 뭐라 했어" → 못 봄"팀즈에서 뭐라 했어" → 메시지 읽기
"노션에서 찾아줘" → 불가"노션에서 찾아줘" → 검색+읽기

오늘 연결할 3개

  • 📮 다우오피스 — 메일 + 캘린더 (회사 기본 도구)
  • 💬 Teams — 채널 메시지, 1:1 채팅
  • 📓 Notion — 문서, 데이터베이스, 위키

이 3개만 연결하면 주요 업무 도구 대부분에 AI가 접근할 수 있어.

⚠️ MCP를 만들 때 반드시 "로컬 MCP"로 만들어야 해. 안 그러면 계정에 종속되는 MCP를 계속 안내하거든. "로컬 MCP로 만들어줘"라고 꼭 말하자.

💡 연동 난이도: Notion(쉬움, 공식 MCP 제공) > Teams(보통) > 다우오피스(어려움). 다우오피스와 Teams는 캠프에서 제공하는 설치 스킬(/setup-daouoffice-mcp, /setup-teams-mcp)을 쓰면 훨씬 수월해.

  1. 1
    Claude Code 터미널에서 claude mcp list 를 입력해서 현재 연결된 MCP가 있는지 확인해
  2. 2
    아마 아무것도 없을 거야. 이제부터 하나씩 연결할 거야!

MCP를 가장 잘 설명한 것은?

A AI를 더 똑똑하게 만드는 업그레이드
B AI가 외부 도구에 접근할 수 있게 해주는 연결 규격
C 새로운 프로그래밍 언어
D 보안 프로토콜
✅ 정답: B. MCP는 AI가 외부 도구(메일, 캘린더, 팀즈 등)에 접근할 수 있게 해주는 연결 규격이야. AI 자체를 똑똑하게 만드는 게 아니라, AI가 "할 수 있는 것"의 범위를 넓혀주는 거야.

2 📮 다우오피스 연결 — 메일 + 캘린더

▼

다우오피스는 ABLE Labs의 기본 업무 도구야. 여기에 메일과 캘린더가 있어. MCP로 연결하면:

할 수 있는 것예시
📬 최근 메일 확인"읽지 않은 메일 보여줘"
🔍 메일 검색"삼성바이오로직스에서 온 메일 찾아줘"
✉️ 메일 발송"이 내용으로 A사에 메일 보내줘"
📅 오늘 일정 확인"오늘 미팅 뭐 있어?"
📆 이번 주 일정"이번 주 스케줄 정리해줘"

연결 방법

Claude Code 터미널에서 이렇게 말하면 돼:

다우오피스 MCP 연결해줘. 로컬 MCP로 만들어줘.

⚠️ "로컬 MCP"로 만들어야 해! 안 그러면 계정에 종속되는 MCP가 만들어져서, 다른 프로젝트에서 중복 설정하거나 꼬일 수 있어. 반드시 "로컬 MCP로 만들어줘"라고 명시하자.

Claude가 필요한 설정을 안내해줘. 프로젝트 폴더의 .mcp.json 파일에 설정이 추가돼.

💡 다우오피스는 공식 MCP가 없어서 설정이 좀 복잡한 편이야. 캠프에서 제공하는 설치 스킬을 쓰면 쉬워:
Claude Code에서 /setup-daouoffice-mcp 를 실행하면 자동으로 설정해줘.

  1. 1
    Claude에게 "다우오피스 MCP 연결해줘"라고 말해
  2. 2
    연결이 되면 "최근 메일 5개 보여줘" 라고 시켜봐
  3. 3
    "오늘 일정 뭐야?" 라고 물어봐
  4. 4
    "이번 주 일정 정리해줘" 도 해봐

💡 처음 연결할 때 인증(로그인)이 필요할 수 있어. Claude가 안내하는 대로 따라하면 돼.

다우오피스 MCP로 할 수 있는 것을 모두 고르면?

A 메일 검색만
B 메일 검색 + 일정 확인
C 메일 검색 + 일정 확인 + 메일 발송
D 일정 확인만
✅ 정답: C. 다우오피스 MCP는 메일 읽기/검색, 캘린더 조회는 물론 메일 발송까지 가능해. AI가 초안을 써주면 확인 후 MCP를 통해 바로 보낼 수 있어.

3 💬 Teams 연결 — 채널 + 채팅

▼

Teams는 사내 소통 도구야. MCP로 연결하면:

할 수 있는 것예시
📋 팀/채널 목록 보기"우리 팀즈 채널 뭐 있어?"
💬 채널 메시지 읽기"general 채널 최근 메시지 보여줘"
✍️ 메시지 보내기"이 내용을 채널에 공유해줘"
🔔 멘션 확인"나를 멘션한 메시지 있어?"
🔍 메시지 검색"NOTABLE 관련 메시지 찾아줘"

⚠️ 중요: Teams 메시지 보내기는 진짜로 전송돼! 전체 채널에 보이니까, 보내기 전에 반드시 내용을 확인해. 처음엔 읽기만 연습하자.

연결 방법

Claude Code 터미널에서:

Teams MCP 연결해줘. 로컬 MCP로 만들어줘.

⚠️ 여기서도 "로컬 MCP"로 만들어야 해! 잊지 마.

Claude가 Microsoft 인증 과정을 안내해줘. 안내에 따라 로그인하면 자동으로 연결돼.

💡 Teams도 설정이 복잡한 편이야. 캠프 설치 스킬을 쓰면 간편해:
Claude Code에서 /setup-teams-mcp 를 실행해봐.

  1. 1
    Claude에게 "Teams MCP 연결해줘"라고 말해
  2. 2
    "우리 팀즈 팀 목록 보여줘" (읽기부터!)
  3. 3
    "general 채널 최근 메시지 5개 보여줘"
  4. 4
    "나를 멘션한 메시지 있어?"

💡 메시지 전송은 Day 2 마무리에서 연습해. 지금은 읽기만!

Teams MCP를 쓸 때 가장 주의해야 할 것은?

A 인터넷 연결 상태
B 메시지 전송 전 반드시 내용 확인
C 하루 사용량 제한
D 영어로만 검색 가능
✅ 정답: B. Teams 메시지는 전송하면 진짜로 전체 채널에 보여. AI가 작성한 내용을 반드시 확인하고 보내야 해. "보내줘"라고 하기 전에 항상 내용을 먼저 확인해.

4 📓 Notion 연결 — 문서 + 데이터베이스

▼

Notion은 회사의 지식 저장소야. MCP로 연결하면:

할 수 있는 것예시
🔍 페이지 검색"NOTABLE 관련 문서 찾아줘"
📖 페이지 읽기"이 페이지 내용 요약해줘"
✏️ 페이지 생성/수정"회의록 페이지 만들어줘"
📊 데이터베이스 조회"태스크 DB에서 내 할 일 보여줘"
💬 코멘트 추가"이 페이지에 코멘트 달아줘"

💡 Notion은 공식 MCP를 제공하고 있어서 연동이 비교적 쉬운 편이야!

연결 방법 (OAuth 로그인 방식)

Notion MCP는 API 키를 직접 만들지 않아도 돼. 계정 로그인(OAuth) 방식으로 연결해.

  1. Claude에게 MCP 연결 요청: "Notion MCP 연결해줘. 로컬 MCP로 만들어줘."
  2. 로그인 안내 따르기: 브라우저에서 Notion 계정으로 로그인 → 접근 허용
  3. 연결 확인: "노션에서 뭐든 검색해봐" 로 테스트

⚠️ 중요: Notion MCP는 API 키가 아닌 OAuth 로그인 방식이라, Claude에게 그냥 "Notion 연결해줘"라고만 하면 진행이 안 될 수 있어. 아래 Teams 채널에 공유된 연동 가이드 텍스트를 복사해서 Claude에게 붙여넣기하면 바로 진행돼.

💡 연동 가이드 텍스트는 Teams #AI-얼리버드 채널에 고정 메시지로 올라가 있어 (캠프 시작 시 제공).

  1. 1
    Teams #AI-얼리버드 채널에서 Notion MCP 연동 가이드 텍스트를 복사해
  2. 2
    Claude Code에 붙여넣기 + "이 방법으로 Notion MCP 연결해줘. 로컬 MCP로 만들어줘."
  3. 3
    브라우저에서 Notion 로그인 → 접근 허용
  4. 4
    연결되면 "노션에서 [키워드] 검색해줘" 해봐
  5. 5
    "테스트 페이지 하나 만들어줘" 로 쓰기도 테스트해봐

💡 Notion Integration은 연결한 페이지만 접근 가능해. 접근하고 싶은 페이지에 직접 Integration을 연결해야 해.

Notion MCP 연결 후에도 특정 페이지에 접근이 안 되는 이유는?

A API 키가 만료돼서
B 그 페이지에 Integration을 연결 안 해서
C 유료 기능이라서
D Notion 서버 문제
✅ 정답: B. Notion은 보안상 Integration을 각 페이지/데이터베이스에 직접 연결해야 접근 가능해. 전체 워크스페이스에 자동 접근은 안 돼.

5 🔀 3개 조합 — 진짜 힘은 여기서

▼

MCP 하나만 써도 유용하지만, 3개를 동시에 쓰면 차원이 달라져.

💡 "오늘의 업무 요약해줘" 한마디면 → 📅 캘린더에서 오늘 일정 확인 → 📬 관련 메일 확인 → 📓 노션에서 태스크 확인 → 종합해서 정리

실전 프롬프트 예시

오늘의 업무를 종합 정리해줘.
1. 오늘 캘린더 일정 확인
2. 최근 중요 메일 확인
3. 노션 태스크 확인
4. 위 정보를 종합해서 투두 리스트 만들어줘

더 써볼 수 있는 것들

  • "이번 주 위클리 리포트 초안 써줘" — 일정 + 메일 + 노션 종합
  • "[고객명] 미팅 준비해줘" — 메일 히스토리 + 노션 문서 + 일정 확인
  • "팀즈에서 나한테 온 요청 정리해줘" — 멘션 + 메일 크로스 체크
  1. 1
    Claude에게 "오늘의 업무를 종합 정리해줘" 라고 시켜봐
  2. 2
    3개 MCP가 동시에 돌아가는 걸 확인해. 캘린더, 메일, 노션을 한번에 조회하는 거야
  3. 3
    결과물이 마음에 들면, "이걸 Teams #AI-얼리버드 채널에 공유해줘" (전송 전 내용 확인!)

💡 3개 MCP를 한번에 쓰는 게 이 캠프의 핵심이야. Day 3부터는 이걸 자동화하는 법을 배워.
