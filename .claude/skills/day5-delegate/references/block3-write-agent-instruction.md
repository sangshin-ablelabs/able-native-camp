# Block 3: 에이전트 인스트럭션 작성

## EXPLAIN

에이전트에게 위임하려면 **인스트럭션**이 필요해. CLAUDE.md에 에이전트가 참고할 지침을 써야 해.

### 에이전트 인스트럭션의 4요소

```markdown
## [업무명] 에이전트 인스트럭션

### 1. 목표 (Goal)
- 이 업무의 최종 목표가 뭔지 명확히
- "~를 완성해", "~를 판단해" 형태

### 2. 사용 가능한 도구 (Available Tools)
- 어떤 MCP를 쓸 수 있는지
- 어떤 스킬을 호출할 수 있는지
- 어떤 파일/폴더에 접근할 수 있는지

### 3. 판단 기준 (Judgment Criteria)
- 우선순위를 어떻게 정할지
- "긴급"의 기준은 뭔지
- 모호할 때 어떻게 할지

### 4. 제약 조건 (Constraints)
- 하면 안 되는 것
- 반드시 사람에게 확인받아야 하는 것
- 형식/톤 제한
```

### 예시: 주간 보고서 에이전트

```markdown
## 주간 보고서 에이전트

### 목표
이번 주 업무 현황을 종합해서 주간 보고서 초안을 완성해.

### 사용 가능한 도구
- Notion MCP: 이번 주 태스크 상태 조회
- Teams MCP: 팀 채널 주요 논의사항 조회
- 메일 MCP: 외부 커뮤니케이션 이력 조회
- Read/Write: 보고서 파일 작성

### 판단 기준
- "성과"는 완료된 태스크 + 수치화된 결과 우선
- "이슈"는 일정 지연, 리소스 부족, 외부 의존성 순으로 중요
- 정보가 부족하면 "확인 필요"로 표시하고 넘어가

### 제약 조건
- 보고서는 A4 1장 분량을 넘지 마
- 개인 평가/인사 관련 내용은 포함하지 마
- 초안 완성 후 반드시 사용자에게 확인 받아
```

### 왜 인스트럭션이 중요한가

인스트럭션 없이 "주간 보고서 만들어줘"라고 하면:
- 어디서 데이터를 가져올지 몰라
- 뭐가 중요하고 뭐가 아닌지 몰라
- 어디서 멈춰야 할지 몰라

인스트럭션이 있으면:
- 판단 기준이 명확해
- 도구 선택이 빨라
- 결과 품질이 일정해

**CLAUDE.md가 에이전트의 두뇌라면, 인스트럭션은 에이전트의 업무 매뉴얼이야.**

## EXECUTE

1. Block 2에서 골라낸 위임 가능 업무 1개를 선택해
2. Claude에게 이렇게 말해봐:

```
[업무 이름]을 에이전트에게 위임하기 위한 인스트럭션을 작성해줘.

목표: [최종 목표]
사용 가능한 도구: [MCP, 스킬 목록]
판단 기준: [어떤 기준으로 판단해야 하는지]
제약 조건: [하면 안 되는 것, 확인 받아야 하는 것]

이걸 CLAUDE.md에 추가할 수 있는 형태로 만들어줘.
```

3. 만들어진 인스트럭션을 CLAUDE.md에 추가해
4. 실제로 에이전트 모드로 실행해봐:

```
[업무 이름] 해줘. 인스트럭션에 따라서 진행하고,
판단한 내용과 근거를 같이 알려줘.
```

5. 결과를 확인하고, 인스트럭션에 보완할 부분이 있으면 수정해

---
위 내용을 직접 실행해봐.
실행이 끝나면 "완료" 또는 "다음"이라고 입력해.
