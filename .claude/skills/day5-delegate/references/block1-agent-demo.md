# Block 1: 에이전트 위임 데모

## EXPLAIN

실제로 에이전트에게 위임하는 사례를 봐.

### 사례: "이번 주 놓친 게 없는지 확인해줘"

이 한마디로 에이전트가 하는 일:

```
[에이전트의 판단 과정]

1. "놓친 게 없는지" → 메일, Teams, 캘린더를 모두 확인해야겠지
   → 캘린더 MCP로 이번 주 일정 조회
   → 메일 MCP로 미회신 메일 조회
   → Teams MCP로 미확인 멘션 조회

2. 일정 중 "준비가 필요한데 아직 안 한 것" 판별
   → 내일 고객 미팅인데 제안서가 아직 없어 → 플래그

3. 미회신 메일 중 "긴급도 높은 것" 판별
   → 발신자, 제목, 날짜를 보고 우선순위 판단

4. Teams 멘션 중 "응답이 필요한 것" 판별
   → 질문형 멘션 vs 단순 공유 구분

5. 종합 리포트 생성
   → 긴급/중요/참고로 3단계 분류
   → 각 항목에 추천 액션 제시
```

### 하네스와의 차이

하네스였다면 이 모든 순서를 사람이 미리 정해줘야 해.
에이전트는 **"놓친 게 없는지 확인해줘"** 한마디에 알아서 판단하는 거야.

특히 중요한 건:
- **우선순위 판단** — 뭐가 긴급하고 뭐가 아닌지
- **예외 처리** — 데이터가 없거나 모호할 때의 대응
- **맥락 활용** — CLAUDE.md에 있는 업무 정보를 참고해서 판단

### 에이전트가 잘 동작하려면

1. **CLAUDE.md가 충실해야 해** — 에이전트의 판단 기준이 여기서 나와
2. **MCP가 연결되어 있어야 해** — 에이전트의 팔다리지
3. **목표가 명확해야 해** — "좀 해줘"보다 "놓친 게 없는지 확인해줘"가 나아

## EXECUTE

1. 에이전트 모드를 체험해봐
2. Claude에게 복잡한 업무 하나를 **목표만 주고** 시켜봐:

```
[업무 목표를 한 문장으로]

방법은 네가 판단해서 진행해줘.
어떤 도구를 쓸지, 어떤 순서로 할지 네가 정해.
중간에 판단이 필요한 부분은 근거도 같이 알려줘.
```

3. 에이전트가 어떤 순서로 어떤 판단을 하는지 관찰해봐
4. 결과를 확인하고 피드백해봐
