# Block 1: CLAUDE.md — AI에게 주는 내 사용설명서

## EXPLAIN

CLAUDE.md는 프로젝트 루트에 놓는 마크다운 파일이다. Claude Code가 대화를 시작할 때 **자동으로 읽는다**.

여기에 뭘 쓰느냐에 따라 AI가 나를 얼마나 잘 아는지가 달라진다.

### 비유

신입이 들어왔다고 생각해봐. 아무것도 안 알려주면 매번 같은 질문을 한다.
하지만 "내 업무는 이거고, 이런 식으로 일하고, 이건 주의해"라고 문서를 줘놓으면?
처음부터 맥락을 가지고 일한다.

CLAUDE.md가 그 문서다.

### 뭘 쓰면 되나

```markdown
# About Me

- 이름: [이름]
- 팀: [팀명]
- 역할: [담당 업무 설명]
- 선호 스타일: [예: 간결하게, 표로 정리, 평어 사용 등]

# My Work

- 주요 업무: [매일/매주 하는 일]
- 자주 쓰는 도구: [다우오피스, Teams, Notion 등]
- 주의사항: [예: 고객명은 영문으로, 금액은 만원 단위로 등]

# ABLE Labs Context

- 회사: ABLE Labs (에이블랩스) — 바이오 실험 자동화 스타트업
- 제품: NOTABLE (액체 핸들링 로봇), NOTABLE96, SUITABLE
- 핵심 고객: 삼성바이오로직스, LG화학, KAIST
- 구성원 호칭: 에이블러 (ABLER)
- 문화: 평어 문화, 코어타임 (7~10시 자유 출근)
```

### CEO의 CLAUDE.md

CEO 신상의 CLAUDE.md에는 이런 것들이 들어있다:
- 제품 라인업과 가격
- 브랜드 표기 규칙 (ABLE Labs, NOTABLE 등 대문자)
- LinkedIn 글쓰기 스타일 가이드
- 의사결정 프로토콜
- 120+ 스킬 연결 정보

에이블러 각자가 자기 버전을 만들면 된다. 처음엔 간단하게, 점점 풍부하게.

## EXECUTE

1. 자기 프로젝트 폴더(또는 홈 디렉토리)에서 Claude Code를 연다
2. Claude에게 이렇게 말한다:

```
CLAUDE.md 파일을 만들어줘.
나는 [이름]이고, ABLE Labs [팀]에서 [역할]을 맡고 있어.
주로 하는 업무는 [업무 설명]이야.
```

3. Claude가 CLAUDE.md 초안을 만들어준다
4. 내용을 확인하고, 추가하고 싶은 내용을 말한다
5. 완성된 CLAUDE.md를 저장한다
