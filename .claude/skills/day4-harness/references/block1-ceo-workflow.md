# Block 1: 데일리 브리핑 워크플로우

## EXPLAIN

매일 아침 하는 데일리 브리핑, 이게 하네스의 실제 사례야.

### 워크플로우 구조

```
[Step 1] 캘린더 MCP → 오늘 일정 조회
    ↓ (일정 목록)
[Step 2] 메일 MCP → 미확인 메일 조회
    ↓ (메일 목록)
[Step 3] Teams MCP → 멘션/중요 메시지 조회
    ↓ (메시지 목록)
[Step 4] 요약 스킬 → 일정 + 메일 + 메시지를 종합 브리핑으로
    ↓ (브리핑 텍스트)
[Step 5] Notion MCP → 투두 카드 1개로 체크리스트 생성
```

### 핵심 포인트

1. **각 단계의 출력이 다음 단계의 입력이 돼** — 이게 하네스야
2. **MCP 3개 + 스킬 1개 + MCP 1개** — 서로 다른 도구들이 하나의 흐름으로
3. **한마디로 실행** — "데일리 브리핑 해줘"

### 왜 이게 중요해

수동으로 하면:
- 다우오피스 열어서 캘린더 확인 (2분)
- 메일함 열어서 스캔 (5분)
- Teams 열어서 읽기 (5분)
- 머릿속에서 종합 (3분)
- 노션에 정리 (5분)
- **총 20분**

하네스로 하면:
- "데일리 브리핑 해줘" (1분)
- 결과 확인하고 조정 (2분)
- **총 3분**

매일 17분 절약. 한 달이면 약 6시간이야.

### 이게 가능한 이유

Day 2에서 MCP를 연결했고, Day 3에서 스킬을 만들었잖아.
부품이 준비됐으니, 이제 엮기만 하면 돼.
