# Block 0: 하네스란 뭐야

## EXPLAIN

Day 3까지 스킬을 만들었지. 스킬 하나하나는 잘 돌아가.

근데 실제 업무는 스킬 1개로 안 끝나잖아. 여러 스킬이 **순서대로 연결**돼야 해.

### 하네스의 정의

> **스킬 A의 출력이 스킬 B의 입력이 되는 거.** 이게 하네스야.

### 비유

자동차를 생각해봐.
- 엔진, 변속기, 바퀴 — 각각은 부품(스킬)이야
- 와이어링 하네스 — 이것들을 연결해서 하나의 시스템으로 만드는 거지

**개별 스킬은 부품이고, 하네스가 시스템을 만들어.**

### Day 1~3 복습

```
Day 1: Operate  → 도구를 쓸 수 있어 (부품을 알아)
Day 2: Connect  → 외부 서비스를 연결해 (부품을 조달해)
Day 3: Automate → 반복 작업을 스킬로 만들어 (부품을 만들어)
Day 4: Harness  → 스킬을 엮어 워크플로우를 만들어 (시스템을 만들어) ← 오늘
```

### 하네스 전 vs 후

**하네스 없이:**
1. "캘린더 확인해줘" → 결과 읽고
2. "이 일정들에 맞는 메일 찾아줘" → 결과 읽고
3. "이걸 요약해줘" → 결과 읽고
4. "노션에 올려줘" → 직접 확인

**하네스 있으면:**
1. "데일리 브리핑 해줘" → 끝

한마디에 4단계가 자동으로 흘러가. 이게 하네스의 힘이야.

## EXECUTE

1. 매일 하는 업무 중 **2단계 이상 거치는 것** 1개를 떠올려봐
2. 그 업무의 단계를 메모해 (예: "메일 확인 → 정리 → 보고" 같은 식)
3. Claude에게 말해봐:

```
나는 매일 [업무 이름]을 할 때 이런 순서로 해:
1단계: [...]
2단계: [...]
3단계: [...]
이걸 하네스로 묶으면 어떻게 될지 설명해줘.
```

---
위 내용을 직접 실행해봐.
실행이 끝나면 "완료" 또는 "다음"이라고 입력해.
