# Block 0: 하네스란 무엇인가

## EXPLAIN

Day 3까지 스킬을 만들었다. 스킬 하나하나는 잘 동작한다.

근데 실제 업무는 스킬 1개로 끝나지 않는다. 여러 스킬이 **순서대로 연결**되어야 한다.

### 하네스의 정의

> **스킬 A의 출력이 스킬 B의 입력이 되는 것.** 이게 하네스다.

### 비유

자동차를 생각해봐.
- 엔진, 변속기, 바퀴 — 각각은 부품(스킬)이다
- 와이어링 하네스 — 이것들을 연결해서 하나의 시스템으로 만드는 것

**개별 스킬은 부품이고, 하네스가 시스템을 만든다.**

### Day 1~3 복습

```
Day 1: Operate  → 도구를 쓸 수 있다 (부품을 안다)
Day 2: Connect  → 외부 서비스를 연결한다 (부품을 조달한다)
Day 3: Automate → 반복 작업을 스킬로 만든다 (부품을 만든다)
Day 4: Harness  → 스킬을 엮어 워크플로우를 만든다 (시스템을 만든다) ← 오늘
```

### 하네스 전 vs 후

**하네스 없이:**
1. "캘린더 확인해줘" → 결과 읽고
2. "이 일정들에 맞는 메일 찾아줘" → 결과 읽고
3. "이걸 요약해줘" → 결과 읽고
4. "노션에 올려줘" → 직접 확인

**하네스 있으면:**
1. "데일리 브리핑 해줘" → 끝

한마디에 4단계가 자동으로 흘러간다. 이게 하네스의 힘이다.

## EXECUTE

1. 자기가 매일 하는 업무 중 **2단계 이상 거치는 것** 1개를 떠올린다
2. 그 업무의 단계를 메모한다 (예: "메일 확인 → 정리 → 보고" 같은 식)
3. Claude에게 말한다:

```
나는 매일 [업무 이름]을 할 때 이런 순서로 해:
1단계: [...]
2단계: [...]
3단계: [...]
이걸 하네스로 묶으면 어떻게 될지 설명해줘.
```
