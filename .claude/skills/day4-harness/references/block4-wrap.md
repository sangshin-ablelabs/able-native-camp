# Block 4: Day 4 마무리

## EXPLAIN

Day 4에서 배운 것:

```
1. 하네스 = 스킬 A의 출력이 스킬 B의 입력이 되는 것
2. 개별 스킬은 부품, 하네스가 시스템을 만든다
3. 워크플로우 설계 = 반복 업무를 단계별로 쪼개고 연결 포인트 찾기
4. 2~3 단계 워크플로우부터 시작해서 점진적으로 확장
```

### 오늘의 미션

Teams #able-native-camp 채널에 아래를 공유한다:

```
[Day 4 완료]
- 만든 워크플로우: [워크플로우 이름]
- 연결한 것: [스킬/MCP 목록]
- 절약 효과: [예: 매일 15분 → 3분]
```

### 6일 여정 현황

```
Day 0: Why      ✅ 왜 해야 하는지 안다
Day 1: Operate  ✅ 도구를 쓸 수 있다
Day 2: Connect  ✅ 외부 서비스를 연결했다
Day 3: Automate ✅ 반복 작업을 스킬로 만들었다
Day 4: Harness  ✅ 스킬을 엮어 워크플로우를 만들었다 ← 오늘
Day 5: Delegate ⬜ 에이전트에게 위임한다
Day 6: Sustain  ⬜ 복리로 성장하는 환경
```

### 내일 예고

Day 5에서는 **에이전트에게 위임**한다.
스킬은 "이거 해줘", 하네스는 "이 순서로 해줘"였다면,
에이전트는 **"이 목표를 달성해줘, 방법은 네가 정해"**다.

단순 실행이 아니라, AI가 판단까지 하게 된다.

```

```
