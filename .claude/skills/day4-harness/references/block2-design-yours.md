# Block 2: 내 워크플로우 설계

## EXPLAIN

이제 내 업무를 워크플로우로 설계해본다.

### 핵심 질문

> **"내가 매주 하는 일의 흐름을 그려봐."**

### 설계 방법

#### Step 1: 반복 업무 뽑기

매일 또는 매주 반복하는 업무를 3개 적어본다.

#### Step 2: 각 업무의 단계 분해

업무 하나를 골라서 단계별로 쪼갠다.

**예시 — 주간 보고서 작성:**
```
1단계: 이번 주 완료 업무 정리 (Notion에서 확인)
2단계: 주요 이슈/리스크 정리 (Teams 메시지에서 확인)
3단계: 다음 주 계획 작성
4단계: 보고서 양식에 맞게 정리
5단계: 팀장에게 메일 발송
```

**예시 — 생산 일보 정리:**
```
1단계: 당일 생산 데이터 수집
2단계: 불량 건수/사유 정리
3단계: 전일 대비 비교
4단계: 일보 양식에 기입
5단계: 공유 폴더에 저장
```

**예시 — 고객 문의 대응:**
```
1단계: 메일에서 고객 문의 확인
2단계: 제품 스펙/FAQ에서 답변 찾기
3단계: 답변 초안 작성
4단계: 관련 자료 첨부
5단계: 답장 발송
```

#### Step 3: 연결 포인트 찾기

각 단계 사이에 "어떤 데이터가 넘어가는지" 화살표로 표시한다.

```
[Step 1] → (완료 업무 목록) → [Step 2] → (이슈 목록) → [Step 3] ...
```

이 화살표가 바로 하네스다.

### 주의: 처음부터 완벽하지 않아도 된다

2~3단계짜리 짧은 워크플로우면 충분하다.
나중에 단계를 추가하면 된다.

## EXECUTE

1. Claude에게 이렇게 말한다:

```
내가 매주 하는 반복 업무를 워크플로우로 설계해줘.

업무: [업무 이름]
순서:
1단계: [...]
2단계: [...]
3단계: [...]

각 단계에서 어떤 데이터가 다음 단계로 넘어가는지,
어떤 MCP나 도구가 필요한지 정리해줘.
```

2. Claude가 만든 워크플로우 다이어그램을 확인한다
3. 빠진 단계나 수정할 부분을 알려준다
4. 최종 워크플로우를 파일로 저장한다
