# Block 3: 워크플로우 직접 만들기

## EXPLAIN

Block 2에서 설계한 워크플로우를 실제로 구축해봐.

### 구축 방법

Day 3에서 만든 스킬 + Day 2에서 연결한 MCP + 새로운 스킬을 조합해.

### 워크플로우 스킬 파일 구조

워크플로우도 결국 스킬 파일이야. 다만 내부에서 **다른 스킬과 MCP를 순서대로 호출**하는 거지.

```markdown
# 주간 보고서 작성 워크플로우

이 스킬이 호출되면 아래 순서로 실행해.

## Step 1: 데이터 수집
- Notion MCP에서 이번 주 완료 태스크 조회
- 결과를 `completed_tasks` 변수로 저장

## Step 2: 이슈 수집
- Teams MCP에서 이번 주 팀 채널 메시지 중 이슈/리스크 키워드 검색
- 결과를 `issues` 변수로 저장

## Step 3: 보고서 생성
- `completed_tasks` + `issues`를 조합해서 주간 보고서 양식에 맞게 작성
- 다음 주 계획은 진행 중인 태스크에서 추출

## Step 4: 저장 및 공유
- 보고서를 마크다운 파일로 저장
- (선택) 메일로 발송
```

### 핵심 원칙

1. **2~3 단계로 시작해** — 처음부터 5단계 워크플로우 만들지 마
2. **각 단계의 입출력을 명확히 해** — "뭘 받아서 뭘 내보내는지" 써놔
3. **수동 확인 포인트를 둬도 돼** — 완전 자동이 아니어도 괜찮아

### 예시: 간단한 2단계 워크플로우

```markdown
# 미확인 메일 요약 워크플로우

## Step 1: 메일 조회
- 메일 MCP로 오늘 수신 메일 중 미확인 메일 조회

## Step 2: 요약
- 각 메일을 1줄로 요약
- 긴급도 기준으로 정렬 (발신자, 키워드 기반)
- 결과를 표 형태로 출력
```

이것만으로도 충분한 워크플로우야. 매일 메일 확인하는 시간이 확 줄어들어.

## EXECUTE

1. Block 2에서 설계한 워크플로우를 실제 스킬 파일로 만들어봐
2. Claude에게 말해봐:

```
Block 2에서 설계한 [업무 이름] 워크플로우를 스킬 파일로 만들어줘.

워크플로우:
Step 1: [...]  → 사용할 도구: [MCP/스킬명]
Step 2: [...]  → 사용할 도구: [MCP/스킬명]
Step 3: [...]  → 사용할 도구: [MCP/스킬명]

각 단계의 입출력이 명확하게 연결되도록 만들어줘.
```

3. 만들어진 스킬 파일을 `.claude/skills/` 아래에 저장해
4. 실제로 실행해서 동작하는지 확인해봐
5. 안 되는 단계가 있으면 수정해
