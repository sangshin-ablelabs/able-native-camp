# Block 0: MCP가 뭐야?

## EXPLAIN

MCP(Model Context Protocol)는 **AI의 손**이다.

Day 1에서 Claude Code는 파일을 읽고, 쓰고, 명령어를 실행할 수 있었다.
하지만 내 이메일을 읽거나, Teams 메시지를 보내거나, Notion 페이지를 만드는 건 못 했다.

왜? 그 도구들에 "연결"이 안 되어 있으니까.

MCP는 그 연결을 만들어주는 규격이다.

### 비유

스마트폰을 생각해봐.
- 폰 자체로는 전화, 문자만 된다
- 앱을 깔면? 카카오톡, 슬랙, 구글캘린더, 뱅킹 다 된다
- 앱 하나 깔 때마다 할 수 있는 일이 늘어난다

MCP는 Claude Code의 "앱스토어"다.
**연결할수록 AI가 할 수 있는 일이 늘어난다.**

### MCP 연결 전 vs 후

| | 연결 전 | 연결 후 |
|---|---------|---------|
| 이메일 | "메일 확인해봐" → 못 함 | "오늘 온 메일 요약해줘" → 바로 실행 |
| 캘린더 | "오늘 일정 뭐야" → 못 함 | "오늘 미팅 몇 개야" → 바로 조회 |
| Teams | "팀즈에 공유해줘" → 못 함 | "이 내용 채널에 올려줘" → 바로 전송 |
| Notion | "노션에서 찾아줘" → 못 함 | "이번달 회의록 찾아줘" → 바로 검색 |

### 어떻게 연결하나

두 가지 방법이 있다:

**방법 1: 명령어로 추가**
```bash
claude mcp add [서버이름] [실행명령]
```

**방법 2: .mcp.json 파일**
프로젝트 루트에 `.mcp.json` 파일을 만들어서 설정한다:
```json
{
  "mcpServers": {
    "서버이름": {
      "command": "실행명령",
      "args": ["인자1", "인자2"],
      "env": {
        "API_KEY": "키값"
      }
    }
  }
}
```

Day 2에서는 3개의 MCP를 연결한다:
1. **다우오피스** — 이메일 + 캘린더
2. **Microsoft Teams** — 채팅 + 채널
3. **Notion** — 문서 + 데이터베이스

## EXECUTE

1. 터미널에서 Claude Code를 실행한다
2. 아래 명령어를 입력해서 현재 연결된 MCP 목록을 확인한다:

```bash
claude mcp list
```

3. 아직 아무것도 없을 수 있다 — 정상이다. 지금부터 하나씩 연결할 거다.
