# Block 0: MCP가 뭐야?

## EXPLAIN

MCP(Model Context Protocol)는 **AI의 손**이야.

Day 1에서 Claude Code는 파일을 읽고, 쓰고, 명령어를 실행할 수 있었지.
하지만 이메일을 읽거나, Teams 메시지를 보내거나, Notion 페이지를 만드는 건 못 했어.

왜? 그 도구들에 "연결"이 안 되어 있으니까.

MCP는 그 연결을 만들어주는 규격이야.

### 비유

스마트폰을 생각해봐.
- 폰 자체로는 전화, 문자만 돼
- 앱을 깔면? 카카오톡, 슬랙, 구글캘린더, 뱅킹 다 돼
- 앱 하나 깔 때마다 할 수 있는 일이 늘어나지

MCP는 Claude Code의 "앱스토어"야.
**연결할수록 AI가 할 수 있는 일이 늘어나.**

### MCP 연결 전 vs 후

| | 연결 전 | 연결 후 |
|---|---------|---------|
| 이메일 | "메일 확인해봐" → 못 함 | "오늘 온 메일 요약해줘" → 바로 실행 |
| 캘린더 | "오늘 일정 뭐야" → 못 함 | "오늘 미팅 몇 개야" → 바로 조회 |
| Teams | "팀즈에 공유해줘" → 못 함 | "이 내용 채널에 올려줘" → 바로 전송 |
| Notion | "노션에서 찾아줘" → 못 함 | "이번달 회의록 찾아줘" → 바로 검색 |

### 어떻게 연결해

**`.mcp.json` 파일**을 프로젝트 폴더에 만들면 돼. 이 파일이 있으면 Claude Code가 자동으로 인식해:

```json
{
  "mcpServers": {
    "서버이름": {
      "type": "stdio",
      "command": "실행명령",
      "args": ["인자"],
      "env": { "API_KEY": "키값" }
    }
  }
}
```

또는 CLI로 추가할 수도 있어:
```bash
claude mcp add [서버이름] [실행명령] [인자]
```

Day 2에서는 3개의 MCP를 연결해:
1. **다우오피스** — 이메일 + 캘린더
2. **Microsoft Teams** — 채팅 + 채널
3. **Notion** — 문서 + 데이터베이스

> 모르는 게 있으면 Claude에게 "MCP가 뭐야?" 하고 물어봐도 돼. Claude가 자기 기능을 설명해줘.

## EXECUTE

1. 터미널(또는 VS Code)에서 Claude Code를 실행해
2. 아래를 입력해서 현재 연결된 MCP 목록을 확인해봐:

```bash
claude mcp list
```

3. 아직 아무것도 없을 수 있어 — 정상이야. 지금부터 하나씩 연결할 거야.
4. (선택) Claude에게 "MCP가 뭔지 설명해줘"라고 물어봐

---
위 내용을 직접 실행해봐.
실행이 끝나면 "완료" 또는 "다음"이라고 입력해.

## QUIZ

```json
AskUserQuestion({
  "questions": [{
    "question": "MCP를 가장 잘 설명한 비유는?",
    "header": "Block 0 퀴즈",
    "options": [
      {"label": "A. AI의 뇌 — 더 똑똑하게 만들어"},
      {"label": "B. AI의 손 — 연결할수록 할 수 있는 일이 늘어나"},
      {"label": "C. AI의 눈 — 더 잘 볼 수 있게 만들어"},
      {"label": "D. AI의 기억 — 더 많이 기억하게 만들어"}
    ],
    "multiSelect": false
  }]
})
```

정답: B. MCP는 AI의 손이야. 도구를 연결할수록 AI가 직접 실행할 수 있는 일이 늘어나. 이메일, 캘린더, Teams, Notion — 연결 하나당 능력 하나가 추가돼.
