# Block 4: 3개 MCP 조합 — 오늘의 업무 요약

## EXPLAIN

이제 진짜 힘을 보여줄 차례다.

MCP 1개만 써도 편하지만, **여러 개를 조합하면** 완전히 다른 차원이 된다.

### 시나리오: "오늘의 업무 요약"

매일 아침 이런 걸 해야 한다고 생각해봐:
1. 다우오피스에서 오늘 일정 확인
2. 다우오피스에서 답장해야 할 메일 확인
3. Notion에서 진행 중인 할 일 확인
4. 이걸 종합해서 정리
5. Teams에 공유

원래는? 다우오피스 열고, Notion 열고, Teams 열고... 최소 10분.

MCP 3개가 연결된 Claude에게는? **한 마디면 끝난다.**

### 마법의 한 마디

```
오늘의 업무 요약을 만들어줘.
- 다우오피스에서 오늘 일정과 최근 메일 확인
- Notion에서 내 할 일 확인
- 종합해서 정리한 뒤 Teams able-native-camp 채널에 공유
```

Claude가 하는 일:
1. `daou_calendar_today` → 오늘 일정 조회
2. `daou_email_recent` → 최근 메일 조회
3. Notion API → 할 일 DB 쿼리
4. 3개 결과를 종합해서 요약 작성
5. `send_channel_message` → Teams 채널에 공유

**하나의 프롬프트로 3개 도구를 넘나든다.** 이게 MCP 조합의 힘이다.

### 응용 예시

```
이번 주 주간보고 초안 만들어줘.
- 이번 주 캘린더에서 미팅 목록 뽑고
- Teams에서 주요 논의 내용 찾고
- Notion 프로젝트 DB에서 진행 상황 확인해서
- 주간보고 양식에 맞춰 정리해줘
```

```
고객 미팅 준비해줘.
- 캘린더에서 오늘 오후 미팅 정보 확인
- Notion에서 해당 고객 관련 문서 검색
- 메일에서 최근 이 고객과 주고받은 내용 확인
- 미팅 브리핑 노트 만들어줘
```

## EXECUTE

이제 직접 "오늘의 업무 요약"을 만들어본다.

1. Claude에게 아래와 같이 말한다:

```
오늘의 업무 요약을 만들어줘.
1. 다우오피스에서 오늘 일정 확인
2. 다우오피스에서 최근 메일 요약
3. 가능하면 Notion에서 할 일도 확인
4. 종합해서 보기 좋게 정리해줘
```

2. 결과를 확인한다
3. 괜찮으면 "이걸 Teams able-native-camp 채널에 공유해줘"라고 한다
4. **공유 전에 내용을 반드시 확인한다**
