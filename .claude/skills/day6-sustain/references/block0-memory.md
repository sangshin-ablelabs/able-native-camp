# Block 0: 메모리 시스템

## EXPLAIN

Claude Code는 대화가 끝나면 기억을 잃는다. 새 대화를 시작하면 백지 상태다.

근데 방법이 있다. **메모리**다.

### 메모리란

대화에서 배운 것, 피드백, 선호도를 **파일로 저장**해서 다음 대화에서도 참고하게 하는 것이다.

### 메모리의 3가지 레벨

```
Level 1: CLAUDE.md
→ 프로젝트 루트에 놓는 기본 설정 파일
→ "나는 누구고, 어떤 일을 하고, 이런 스타일로 해줘"
→ Day 1에서 이미 만들었다

Level 2: Memory 폴더
→ .claude/ 또는 프로젝트 내 memory/ 폴더에 학습 내용 저장
→ "이 피드백은 기억해줘", "이건 다음에도 이렇게 해줘"
→ 대화를 넘어 축적되는 학습

Level 3: 스킬 파일
→ .claude/skills/ 폴더에 재사용 가능한 스킬 저장
→ Day 3에서 만든 것
→ 나뿐 아니라 팀원도 쓸 수 있다
```

### 메모리가 쌓이면

```
1주차: Claude가 내 이름과 팀을 안다
2주차: Claude가 내 업무 스타일을 안다
1개월: Claude가 내 주요 거래처, 자주 쓰는 양식을 안다
3개월: Claude가 "나에게 최적화된 비서"가 된다
```

이게 **복리 성장**이다. 쓸수록 더 잘 동작한다.

### 메모리 저장하는 법

Claude에게 이렇게 말하면 된다:

```
이걸 기억해줘: [내용]
```

또는 더 구체적으로:

```
이 피드백을 메모리에 저장해줘:
- 금액 표기는 억/만원 단위로 해줘
- 보고서는 항상 표 형태로 만들어줘
- 이메일 답장 톤은 정중하지만 간결하게
```

### CEO의 메모리

CEO 신상의 메모리에는 이런 것들이 쌓여 있다:
- 금액 표기 규칙
- 이메일 톤 선호도
- 계약서 리뷰 방식
- 데일리 브리핑 포맷
- 팀즈 글쓰기 스타일

이걸 처음부터 다 정한 게 아니다. **쓰면서 하나씩 피드백하고 저장**한 것이다.

## EXECUTE

1. 지금까지 Claude를 쓰면서 "이건 다음에도 이렇게 했으면 좋겠다"고 느낀 것 3개를 적는다
2. Claude에게 말한다:

```
아래 내용을 메모리에 저장해줘:

1. [선호/피드백 1]
2. [선호/피드백 2]
3. [선호/피드백 3]
```

3. 저장된 메모리 파일을 확인한다
4. 새 대화를 시작해서 메모리가 반영되는지 테스트한다
