# Block 2: 스킬 공유 세션

## EXPLAIN

6일간 각자 만든 스킬, 워크플로우, 에이전트 인스트럭션을 서로 공유해.

### 왜 공유하는가

- **한 사람이 만든 스킬은 한 사람만 써**
- **공유하면 27명 모두가 쓸 수 있어**
- 개인의 생산성 향상이 조직의 생산성 향상이 되는 거야

### 공유 세션 형식

**현장 발표 (다같이 모일 때):**
```
시간: 1인당 3분
형식:
  1. 내가 만든 것 소개 (1분)
     - 스킬/워크플로우/에이전트 중 하나
  2. 데모 (1분)
     - 실제 실행해서 보여주기
  3. 효과 (30초)
     - 전: 얼마나 걸렸는지
     - 후: 얼마나 단축됐는지
  4. Q&A (30초)
```

**비동기 공유 (각자 시간에):**
```
Teams #able-native-camp 채널에 아래 형식으로 공유:

[스킬 공유]
- 스킬명: /my-skill-name
- 기능: 뭘 하는 건지 한줄
- Before: 이전에 걸리던 시간/수고
- After: 자동화 후
- SKILL.md 첨부 또는 코드 블록으로 붙여넣기
```

> 비동기로 하더라도 팀원들이 직접 설치해서 써볼 수 있도록 SKILL.md를 공유하는 게 핵심이야.

### 베스트 스킬 선정

발표 후 투표해:
- **가장 실용적인 스킬** — 당장 내 업무에도 쓸 수 있는 거
- **가장 창의적인 스킬** — 이런 것도 되는구나 싶은 거
- **가장 큰 시간 절약** — 절약 효과가 가장 큰 거

### 베스트 스킬은 조직 공통 스킬이 돼

선정된 스킬은 전사 `.claude/skills/`에 등록되어 모든 에이블러가 사용할 수 있게 돼.

개인의 노하우가 → 조직의 자산이 되는 거지.

### 발표 준비 팁

1. **화면 공유 준비** — 터미널에서 실행하는 모습을 보여줘
2. **Before/After** — 수치로 보여주면 임팩트가 커
3. **간결하게** — 3분은 짧아. 핵심만 말해

## EXECUTE

1. 발표할 스킬/워크플로우/에이전트 1개를 골라봐
2. Claude에게 발표 준비를 시켜봐:

```
스킬 공유 세션 발표를 준비해줘.

발표할 것: [스킬/워크플로우 이름]
기능: [뭘 하는 건지]
전: [이전에 걸리던 시간/수고]
후: [자동화 후 시간/수고]

3분 발표 스크립트를 만들어줘.
데모 순서도 정리해줘.
```

3. 실제로 데모가 동작하는지 미리 테스트해봐
4. 발표 스크립트를 한번 읽어봐

---

실행이 끝나면 "완료" 또는 "다음"이라고 입력해줘.
