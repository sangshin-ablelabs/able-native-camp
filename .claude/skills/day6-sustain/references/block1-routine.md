# Block 1: 개인 루틴 세팅

## EXPLAIN

도구를 배우는 것과 매일 쓰는 것은 다르다. 매일 쓰려면 **루틴**이 필요하다.

### 루틴이란

"아침에 출근하면 이걸 한다", "금요일 오후에는 이걸 한다" — 이런 습관적 패턴.

Claude Code를 루틴에 넣으면, 매일 자동으로 AI와 일하게 된다.

### 일일 루틴 예시

```
[아침 — 출근 직후]
1. 터미널에서 claude 실행
2. "데일리 브리핑 해줘" (Day 4 워크플로우)
3. 오늘 할 일 확인 + 우선순위 정리

[오전 — 업무 시작]
4. 첫 번째 업무를 Claude와 함께 시작
5. "이 메일 답장 초안 써줘", "이 자료 정리해줘" 등

[오후 — 업무 마무리]
6. 오늘 한 일 정리
7. 내일 할 일 세팅
8. 피드백할 것 있으면 메모리에 저장
```

### 주간 루틴 예시

```
[월요일] 주간 목표 세팅 — Claude와 이번 주 우선순위 정리
[수요일] 중간 점검 — 진행 상황 확인, 지연 업무 알림
[금요일] 주간 리뷰 — 이번 주 성과 정리 + 주간 보고서 초안
```

### 루틴의 핵심

1. **매일 같은 시간에 같은 것으로 시작** — 습관이 된다
2. **작은 것부터** — "데일리 브리핑"하나면 충분하다
3. **점진적 확장** — 1주일에 루틴 1개씩 추가

### CLAUDE.md에 루틴 기록

```markdown
# My Routine

## Daily
- 아침: 데일리 브리핑 (캘린더 + 메일 + Teams 확인)
- 오후: 오늘 마무리 정리

## Weekly
- 금요일: 주간 보고서 초안 작성
```

이렇게 써놓으면 Claude가 "오늘 아침 브리핑 할까?"라고 알아서 맥락을 잡을 수 있다.

## EXECUTE

1. 자기의 일일 루틴을 3단계로 설계한다
2. Claude에게 말한다:

```
내 일일 루틴을 설계해줘.

출근 시간: [시간]
주요 업무: [...]
매일 반복하는 일: [...]

아침/오전/오후로 나눠서,
Claude Code를 어떻게 활용할 수 있는지 포함해서 만들어줘.
이걸 CLAUDE.md의 My Routine 섹션에 추가해줘.
```

3. 설계된 루틴을 CLAUDE.md에 추가한다
4. 내일부터 실제로 실행해본다 (아침 브리핑부터)
