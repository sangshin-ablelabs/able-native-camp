# Block 1: SKILL.md 파일 구조 해부

## EXPLAIN

SKILL.md는 정해진 구조가 있다. 하나씩 뜯어보자.

### 1. 프론트매터 (Front Matter)

파일 맨 위에 `---`로 감싸는 메타 정보다.

```yaml
---
name: weekly-production-report
description: 주간 생산 실적을 정리하는 스킬. "주간생산", "생산실적" 요청에 사용.
---
```

- **name**: 스킬 이름. 슬래시 커맨드가 된다 → `/weekly-production-report`
- **description**: 이 스킬이 뭘 하는지. Claude가 언제 이 스킬을 써야 하는지 판단하는 기준이다.

### 2. 제목과 목적

```markdown
# 주간 생산 실적 보고서

이 스킬이 호출되면 아래 절차에 따라 주간 생산 실적 보고서를 작성한다.
```

한 줄로 이 스킬의 목적을 명확하게 적는다.

### 3. 지시사항 (Instructions)

스킬의 핵심. Claude가 **뭘, 어떤 순서로** 해야 하는지 적는다.

```markdown
## 절차

1. 생산 데이터 파일을 읽는다 (경로: data/production/)
2. 이번 주(월~금) 데이터만 필터링한다
3. 제품별 생산량, 불량률을 계산한다
4. 전주 대비 증감을 표시한다
5. 아래 양식으로 보고서를 작성한다
```

**구체적일수록 좋다.** "보고서 만들어"가 아니라 "어떤 데이터를, 어떤 기준으로, 어떤 형식으로" 적어야 한다.

### 4. 참조 파일 (References) — 선택

```markdown
## References

- `references/report-format.md` — 보고서 양식
- `references/product-codes.md` — 제품 코드 목록
```

스킬 실행에 필요한 추가 정보를 별도 파일로 분리해서 넣는다.

### 5. 템플릿 (Templates) — 선택

```markdown
## 출력 양식

templates/weekly-report.md 파일의 양식을 따른다.
```

결과물의 양식이 정해져 있으면 템플릿으로 만들어놓는다.

### 전체 구조 요약

```
SKILL.md 파일 구조:
┌─────────────────────────┐
│ --- (프론트매터)          │  ← 이름, 설명
│ name / description      │
│ ---                     │
├─────────────────────────┤
│ # 제목                   │  ← 목적
│ 이 스킬은...             │
├─────────────────────────┤
│ ## 절차                  │  ← 핵심: 단계별 지시
│ 1. ...                  │
│ 2. ...                  │
├─────────────────────────┤
│ ## References            │  ← 참고 자료 (선택)
├─────────────────────────┤
│ ## 출력 양식              │  ← 템플릿 (선택)
└─────────────────────────┘
```

### 실제 예시 — 연구팀 실험 보고서 스킬

```yaml
---
name: experiment-report
description: 실험 데이터를 분석해 보고서를 작성하는 스킬. "실험보고서", "데이터분석" 요청에 사용.
---
```

```markdown
# 실험 보고서 작성

이 스킬이 호출되면 아래 절차에 따라 실험 보고서를 작성한다.

## 절차

1. 사용자에게 실험 데이터 파일 경로를 묻는다
2. 데이터를 읽고 기본 통계(평균, 표준편차, n수)를 계산한다
3. 이상치(평균 ± 3SD 밖)를 식별하고 표시한다
4. 결과를 아래 양식으로 작성한다:
   - 실험 목적
   - 방법 요약
   - 결과 (표 + 그래프 설명)
   - 고찰
5. 파일명: YYYYMMDD_실험보고서_[실험명].md

## 주의사항

- 단위는 반드시 SI 단위로 표기한다
- 소수점은 유효숫자 3자리까지
- raw data는 절대 삭제하지 않는다
```

## EXECUTE

1. `templates/skill-template.md` 파일을 열어서 빈 SKILL.md 구조를 확인한다
2. 각 섹션(프론트매터, 제목, 절차, 참조, 템플릿)이 뭘 담당하는지 한번 읽어본다
3. Day 1에서 만든 CLAUDE.md와 비교해봐 — CLAUDE.md는 "나를 알려주는 파일", SKILL.md는 "업무를 가르치는 파일"
