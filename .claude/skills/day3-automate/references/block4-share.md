# Block 4: 동료와 교환 테스트

## EXPLAIN

스킬은 혼자 쓸 수도 있지만, **다른 사람이 써봐야 진짜 완성**이야.

### 왜 교환 테스트인가

자기가 만든 스킬은 자기한테 당연한 맥락이 들어있어. "이건 당연히 알겠지"라고 생각한 부분이 다른 사람한테는 안 통할 수 있거든.

```
예시:
만든 사람: "production/ 폴더에서 데이터 읽어"
다른 사람: "production/ 폴더가 어디 있는지 모르는데?"

→ 경로 안내가 빠져있었지.
```

교환 테스트를 하면:
- 빠진 절차를 발견해
- 설명이 불명확한 부분을 찾아
- "이거 추가하면 더 좋겠다"는 아이디어를 얻어

### 교환 테스트 방법

**현장 진행 (같이 있을 때):**
```
1. 짝을 지어 (옆 사람, 같은 팀, 다른 팀 누구든)
2. 서로의 SKILL.md 파일을 공유해
3. 상대방의 스킬을 내 환경에서 실행해봐
4. 피드백을 줘
5. 받은 피드백을 SKILL.md에 반영해
```

**비동기 진행 (각자 시간에 할 때):**
```
1. Teams #able-native-camp 채널에 내 SKILL.md를 공유해
2. 다른 에이블러의 스킬을 골라서 내 환경에서 실행해봐
3. 피드백을 Teams 스레드에 댓글로 남겨
4. 내 스킬에 달린 피드백을 확인하고 반영해
```

### 피드백 줄 때 체크리스트

```
□ 스킬이 호출됐어?
□ 절차를 따라가면 결과물이 나와?
□ 중간에 막히는 단계가 있어?
□ 결과물이 실제로 쓸만해?
□ 한 가지 개선점을 제안한다면?
```

### 좋은 피드백 예시

```
❌ "잘 됐어" (정보가 없지)

✅ "3번 단계에서 파일 경로가 안 맞아서 멈췄어.
    경로를 사용자한테 물어보는 단계를 추가하면 좋겠어.
    나머지는 잘 돌아가고, 결과물 양식 깔끔해."
```

## EXECUTE

1. 짝을 짓거나, Teams에서 다른 에이블러의 스킬을 골라
2. 상대방의 SKILL.md를 내 `.claude/skills/` 아래에 복사해
3. `/스킬명`으로 실행해봐
4. 위 체크리스트 기준으로 피드백을 전달해 (현장: 구두 / 비동기: Teams 스레드)
5. 받은 피드백을 SKILL.md에 반영해

끝나면 "완료" 또는 "다음"이라고 입력해.
