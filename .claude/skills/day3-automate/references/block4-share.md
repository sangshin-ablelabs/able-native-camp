# Block 4: 동료와 교환 테스트

## EXPLAIN

스킬은 혼자 쓸 수도 있지만, **다른 사람이 써봐야 진짜 완성**된다.

### 왜 교환 테스트인가

자기가 만든 스킬은 자기한테 당연한 맥락이 들어있다. "이건 당연히 알겠지"라고 생각한 부분이 다른 사람한테는 안 통할 수 있다.

```
예시:
만든 사람: "production/ 폴더에서 데이터 읽어"
다른 사람: "production/ 폴더가 어디 있는지 모르는데?"

→ 경로 안내가 빠져있었다.
```

교환 테스트를 하면:
- 빠진 절차를 발견한다
- 설명이 불명확한 부분을 찾는다
- "이거 추가하면 더 좋겠다"는 아이디어를 얻는다

### 교환 테스트 방법

```
1. 짝을 짓는다 (옆 사람, 같은 팀, 다른 팀 누구든)
2. 서로의 SKILL.md 파일을 공유한다
3. 상대방의 스킬을 내 환경에서 실행해본다
4. 피드백을 준다:
   - 잘 되는 부분
   - 안 되거나 헷갈리는 부분
   - 이거 추가하면 좋겠다는 점
5. 피드백 받은 걸 반영해서 수정한다
```

### 피드백 줄 때 체크리스트

```
□ 스킬이 호출됐는가?
□ 절차를 따라가면 결과물이 나오는가?
□ 중간에 막히는 단계가 있는가?
□ 결과물이 실제로 쓸만한가?
□ 한 가지 개선점을 제안한다면?
```

### 좋은 피드백 예시

```
❌ "잘 됐어" (정보가 없다)

✅ "3번 단계에서 파일 경로가 안 맞아서 멈췄어.
    경로를 사용자한테 물어보는 단계를 추가하면 좋겠어.
    나머지는 잘 돌아가고, 결과물 양식 깔끔하다."
```

## EXECUTE

1. 옆 사람과 짝을 짓는다
2. 서로의 SKILL.md 파일을 보여준다 (파일 경로 공유 또는 화면 공유)
3. 상대방의 스킬을 내 Claude Code에서 실행해본다:
   - SKILL.md를 복사해서 내 `.claude/skills/` 아래에 넣고
   - `/스킬명`으로 실행
4. 위 체크리스트 기준으로 피드백을 구두로 전달한다
5. 받은 피드백을 SKILL.md에 반영한다
