# Block 3: 스킬 직접 만들기

## EXPLAIN

Block 2에서 고른 업무를 진짜 스킬로 만든다. 3단계로 진행한다.

### Step 1: 절차 정리

스킬을 만들기 전에 먼저 업무 절차를 말로 풀어본다.

```
예시: "주간 생산 실적 정리" 절차

1. 이번 주 생산 데이터를 production/ 폴더에서 읽는다
2. 제품별(NOTABLE, NOTABLE96, SUITABLE)로 분류한다
3. 각 제품의 생산량, 불량수, 불량률을 계산한다
4. 전주 대비 증감을 표시한다
5. 특이사항(라인 중단, 자재 이슈 등)을 별도 섹션에 적는다
6. 정해진 양식으로 보고서를 만든다
7. 파일명: YYYYMMDD_주간생산실적.md
```

**포인트**: "내가 이 업무를 할 때 머릿속에서 하는 순서"를 그대로 적으면 된다.

### Step 2: SKILL.md 작성

`templates/skill-template.md`를 복사해서 채운다.

```
복사 경로:
.claude/skills/[내-스킬명]/SKILL.md
```

채울 때 체크리스트:

```
□ name: 짧고 명확한가? (영문 추천, 한글도 가능)
□ description: 언제 이 스킬을 쓰는지 명확한가?
□ 절차: 단계별로 구체적인가?
□ 입력: 사용자에게 뭘 물어봐야 하는가?
□ 출력: 결과물 형식이 명확한가?
□ 주의사항: 빠뜨리면 안 되는 규칙이 있는가?
```

### Step 3: 테스트 & 반복

만든 스킬을 바로 실행해본다.

```bash
# Claude Code에서
/내-스킬명
```

테스트 체크리스트:

```
□ 슬래시 커맨드로 호출되는가?
□ 절차를 순서대로 따르는가?
□ 결과물이 원하는 형식인가?
□ 빠진 단계가 있는가?
□ 불필요한 단계가 있는가?
```

안 되는 부분이 있으면 SKILL.md를 수정하고 다시 실행한다. **처음부터 완벽할 필요 없다.** 3~4번 고치면 쓸만해진다.

### 잘 만든 스킬 vs 못 만든 스킬

```
❌ 못 만든 스킬:
"보고서를 작성한다"

→ 어떤 보고서? 어떤 데이터로? 어떤 형식으로? AI가 모른다.

✅ 잘 만든 스킬:
"production/ 폴더의 이번 주 CSV 파일을 읽어서,
제품별 생산량과 불량률을 계산하고,
전주 대비 증감을 포함해 아래 양식으로 작성한다.
파일명은 YYYYMMDD_주간생산실적.md로 한다."

→ 데이터 위치, 계산 방법, 출력 형식, 파일명까지 명확.
```

핵심: **"내가 옆자리 동료에게 이 업무를 인수인계한다면 뭘 알려줘야 할까?"** 그걸 적으면 된다.

## EXECUTE

1. 자기 프로젝트 폴더에 스킬 디렉토리를 만든다:

```bash
mkdir -p .claude/skills/[내-스킬명]
```

2. 템플릿을 복사한다:

```bash
cp [캠프경로]/.claude/skills/day3-automate/templates/skill-template.md .claude/skills/[내-스킬명]/SKILL.md
```

3. SKILL.md를 열고 Block 2에서 정리한 업무 절차를 채운다

4. 저장하고 `/내-스킬명`으로 실행해본다

5. 결과를 보고 SKILL.md를 수정한다 — 최소 2번은 고쳐본다

막히면 Claude에게 물어봐:

```
이 SKILL.md 봐줘. 빠진 거 있어? 더 구체적으로 만들려면 뭘 추가해야 해?
```
