# Block 0: 스킬이란 무엇인가

## EXPLAIN

스킬 = 자주 하는 요청을 한 단어로 실행하는 것.

### 비유

매번 이렇게 말한다고 생각해봐:

```
"지난주 실험 데이터를 엑셀에서 읽어서, 평균값 계산하고,
이상치 빼고, 그래프 만들어서, 보고서 양식에 넣어줘.
아, 그리고 파일명은 YYYYMMDD_실험보고서로 해줘."
```

이걸 매번 치나? 안 친다. 한번 가르쳐놓으면 이렇게만 하면 된다:

```
/실험보고서
```

끝. 스킬은 **AI에게 가르치는 업무 매뉴얼**이다. 한번 만들면 매번 반복 안 해도 된다.

### 스킬의 정체

스킬의 실체는 단순하다 — 마크다운 파일 하나다.

```
.claude/skills/내-스킬명/
├── SKILL.md          ← AI가 읽는 매뉴얼
├── references/       ← 참고 자료 (선택)
└── templates/        ← 템플릿 (선택)
```

SKILL.md에 "이 스킬이 호출되면 이렇게 해라"고 적어놓으면, `/스킬명`을 입력할 때 Claude가 그대로 실행한다.

### 각 팀에서의 예시

| 팀 | 반복 업무 | 스킬로 만들면 |
|----|----------|-------------|
| 제조 | 주간 생산 실적 정리 | `/주간생산` |
| 연구 | 실험 데이터 보고서 작성 | `/실험보고서` |
| 경영지원 | 월말 비용 정산 요약 | `/비용정산` |
| 마케팅 | 뉴스레터 초안 작성 | `/뉴스레터` |
| 영업 | 미팅 후 팔로업 메일 | `/팔로업` |
| QA | 테스트 결과 리포트 | `/QA리포트` |

누구든, 어떤 업무든 스킬로 만들 수 있다. 핵심은 **"내가 반복하는 일"**이라는 점이다.

## EXECUTE

1. 지금까지 Claude Code에 가장 길게 입력했던 요청을 떠올려본다
2. "이걸 매번 치는 게 귀찮다"는 생각이 드는 업무가 있는지 생각한다
3. 옆 사람과 30초만 얘기해봐 — "나는 매주 이거 반복해"
