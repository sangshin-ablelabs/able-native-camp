🖥️ Claude Code 기본

설치

npm install -g @anthropic-ai/claude-code

실행

claude

터미널에서 claude를 입력하면 대화형 세션이 시작돼.

기본 도구 4가지

도구설명예시
Read파일 읽기"이 파일 읽어줘"
Write새 파일 만들기"보고서 파일 만들어줘"
Edit기존 파일 수정"3번째 줄 고쳐줘"
Bash터미널 명령 실행"git status 확인해줘"

💡 자연어로 말하면 돼. "이 엑셀 파일 읽어서 요약해줘" 같은 식으로!

💻 VS Code 연동

Claude Code는 터미널에서도 쓸 수 있지만, VS Code에서 쓰면 파일 편집, 탐색, 터미널이 한 화면에 통합돼서 생산성이 훨씬 높아.

확장 프로그램 설치

VS Code 마켓플레이스에서 Claude Code를 검색하거나, 터미널에서:

code --install-extension anthropic.claude-code

설치 후 사이드바에 Claude 아이콘이 나타나. 클릭하면 채팅 패널이 열려.

VS Code에서 쓰면 뭐가 좋아?

기능터미널 단독VS Code 연동
파일 편집Claude가 수정 → 터미널에서 diff 확인에디터에서 실시간으로 변경사항 확인
파일 탐색경로를 직접 입력탐색기에서 클릭으로 파일 참조 가능
코드 선택전체 파일 단위드래그로 특정 코드 블록만 선택해서 질문
터미널별도 창하단 터미널 통합
Git명령어 직접 입력소스 컨트롤 탭에서 시각적 diff

핵심 사용법

  • 채팅 패널 열기: 사이드바 Claude 아이콘 클릭 또는 Cmd+Shift+P → "Claude Code: Open"
  • 코드 선택 후 질문: 코드를 드래그해서 선택 → 채팅에 자동으로 컨텍스트 전달
  • 인라인 편집 확인: Claude가 파일을 수정하면 에디터에서 바로 diff가 뜸 → Accept/Reject 선택
  • 터미널 연동: Claude가 bash 명령을 실행하면 하단 터미널에서 실시간 확인

추천 워크플로우

// 1. 왼쪽: 파일 탐색기로 프로젝트 구조 파악 // 2. 가운데: 에디터에서 코드 확인 // 3. 오른쪽: Claude 채팅 패널 // 4. 하단: 터미널 💡 이 4분할 레이아웃이면 Claude와 페어 프로그래밍 환경 완성!

💡 비개발자도 OK — "이 파일 뭐하는 건지 설명해줘", "이 데이터 정리해줘" 같은 일상 업무도 VS Code + Claude가 훨씬 편해.

📝 CLAUDE.md 작성법

CLAUDE.md는 AI에게 "나는 누구고, 어떻게 일하는지" 알려주는 파일이야. 프로젝트 루트에 두면 자동으로 읽어.

필수 항목

# CLAUDE.md ## 나는 누구인가 - 이름: [이름] - 팀: [팀명] - 역할: [역할 설명] ## 업무 맥락 - 주요 업무: [업무 설명] - 자주 쓰는 도구: [다우오피스, Teams, Notion 등] ## 선호 스타일 - 답변은 간결하게 - 한국어로 대화 - 평어체 사용

💡 CLAUDE.md는 계속 업데이트해. 쓰다 보면 "이것도 알려줘야겠다" 싶은 게 생겨.

🔌 MCP 연결하기

MCP(Model Context Protocol)로 외부 도구를 AI에 직접 연결해. 연결하면 AI가 실시간 데이터를 조회할 수 있어.

연결 방법

Claude Code 대화 중에 자연어로 요청하면 돼:

"다우오피스 MCP 연결해줘" "Teams MCP 연결해줘" "Notion MCP 연결해줘"

연결 후 가능한 것

MCP할 수 있는 것
다우오피스메일 확인/검색, 일정 조회, 캘린더 파싱
Teams채널 메시지 읽기/보내기, 멘션 확인, 파일 공유
Notion페이지 검색/생성/수정, 데이터베이스 조회

⚠️ MCP 연결 시 인증이 필요해. 각 서비스의 API 키나 OAuth 설정을 미리 준비해두자.

⚡ 스킬 만들기

반복 업무를 /명령어 하나로 실행할 수 있게 만드는 거야.

SKILL.md 기본 구조

# /weekly-report 주간 업무 보고서를 자동으로 생성한다. ## 트리거 - "주간보고", "위클리 리포트", "/weekly-report" ## 실행 단계 1. 이번 주 완료한 업무를 정리한다 2. 다음 주 계획을 작성한다 3. 보고서 포맷에 맞춰 출력한다 ## 출력 형식 - 마크다운 표 형태 - 주요 성과 / 이슈 / 다음 주 계획

스킬 파일 위치

.claude/skills/[스킬이름]/SKILL.md

💡 좋은 스킬을 만들었으면 스킬 허브에 등록해서 동료와 공유하자!

🔗 워크플로우 구성

스킬 A의 출력이 스킬 B의 입력이 되는 멀티스텝 흐름을 만드는 거야.

예시: 데일리 브리핑 워크플로우

Step 1: 메일 확인 (다우오피스 MCP) ↓ 안 읽은 메일 목록 Step 2: 일정 파싱 (다우오피스 MCP) ↓ 오늘 미팅 리스트 Step 3: 팀즈 멘션 확인 (Teams MCP) ↓ 답장 필요한 메시지 Step 4: 투두 생성 (종합) → 오늘의 할 일 체크리스트

워크플로우 설계 팁

  • 2~3단계로 시작해. 너무 복잡하면 디버깅이 어려워.
  • 각 단계의 입력/출력을 명확하게 정의해.
  • 실패 시 대안(fallback)을 생각해둬.

🤖 에이전트 위임

스킬은 "이거 해줘". 에이전트는 "이 목표를 달성해줘, 방법은 네가 정해."

좋은 에이전트 지시문 예시

목표: 이번 주 금요일까지 고객 A사 미팅 준비 완료 해야 할 것: - CRM에서 A사 최근 활동 이력 조회 - 이전 미팅 기록 요약 - 제안할 사항 3가지 정리 - 브리핑 문서 1장으로 만들기 제약: - 내부 정보만 사용 - 본인 확인 후 최종 발송

위임 가능한 업무 체크리스트

  • ✅ 데이터 수집 + 정리 + 보고서 작성
  • ✅ 정기적인 모니터링 + 알림
  • ✅ 여러 소스를 조합한 분석
  • ❌ 최종 의사결정 (사람이 해야 해)
  • ❌ 외부 발송/게시 (확인 후에)

🧠 Memory 시스템

Memory는 대화 간 맥락을 기억하게 해주는 시스템이야. 매번 처음부터 설명할 필요가 없어져.

메모리 저장하기

메모리 종류

종류용도예시

📚 용어 사전

캠프에서 자주 나오는 용어들을 모았어. 모르는 단어가 나오면 여기서 찾아봐.

용어설명
토큰 (Token)AI가 텍스트를 처리하는 단위. 한글 1글자 ≈ 1~2토큰, 영어 1단어 ≈ 1토큰. 토큰을 많이 쓸수록 비용이 올라가. 리더보드의 순위는 토큰 사용량 기준이야.
Git코드/파일의 변경 이력을 관리하는 도구. "저장" 버전을 쌓아두는 거라 생각하면 돼. git clone은 프로젝트를 복사해오는 것, git pull은 최신 변경사항을 가져오는 것.
터미널 (Terminal)텍스트 명령어로 컴퓨터를 조작하는 창. Mac은 "터미널" 앱, Windows는 "PowerShell" 또는 "명령 프롬프트"라고 해. Claude Code는 여기서 실행해.
Node.jsJavaScript를 컴퓨터에서 실행할 수 있게 해주는 프로그램. Claude Code가 돌아가려면 필요해. 한 번만 설치하면 돼.
npmNode.js의 패키지(프로그램) 설치 도구. npm install = 필요한 프로그램을 다운받아 설치하는 것.
MCPModel Context Protocol. AI에게 외부 도구(메일, 캘린더, 팀즈 등)를 연결하는 규격. 스마트폰에 앱 설치하듯, Claude에 MCP를 연결하면 새 능력이 생겨.
로컬 MCP내 프로젝트 폴더에만 적용되는 MCP 설정. 계정에 종속되지 않아서, 프로젝트별로 독립적으로 관리할 수 있어.
CLAUDE.mdAI에게 "나는 누구고, 어떻게 일하는지" 알려주는 설정 파일. 프로젝트 폴더에 넣으면 자동으로 읽어.
스킬 (Skill)반복 업무를 /명령어 하나로 자동 실행하게 만드는 것. SKILL.md 파일로 정의해.
컨텍스트 (Context)AI가 "기억하고 있는" 대화 내용의 범위. 대화가 길어지면 컨텍스트가 커져서 비용이 올라가. /compact로 줄일 수 있어.
프롬프트 (Prompt)AI에게 내리는 지시/질문. 프롬프트를 잘 쓰면 결과물의 질이 확 올라가.
Hook특정 이벤트가 발생하면 자동으로 실행되는 스크립트. 캠프에서는 세션 종료 시 토큰 사용량을 자동 보고하는 Hook을 사용해.
