🖥️ Claude Code 기본

설치

npm install -g @anthropic-ai/claude-code

실행

claude

터미널에서 claude를 입력하면 대화형 세션이 시작돼.

기본 도구 4가지

도구설명예시
Read파일 읽기"이 파일 읽어줘"
Write새 파일 만들기"보고서 파일 만들어줘"
Edit기존 파일 수정"3번째 줄 고쳐줘"
Bash터미널 명령 실행"git status 확인해줘"

💡 자연어로 말하면 돼. "이 엑셀 파일 읽어서 요약해줘" 같은 식으로!

📝 CLAUDE.md 작성법

CLAUDE.md는 AI에게 "나는 누구고, 어떻게 일하는지" 알려주는 파일이야. 프로젝트 루트에 두면 자동으로 읽어.

필수 항목

# CLAUDE.md ## 나는 누구인가 - 이름: [이름] - 팀: [팀명] - 역할: [역할 설명] ## 업무 맥락 - 주요 업무: [업무 설명] - 자주 쓰는 도구: [다우오피스, Teams, Notion 등] ## 선호 스타일 - 답변은 간결하게 - 한국어로 대화 - 평어체 사용

💡 CLAUDE.md는 계속 업데이트해. 쓰다 보면 "이것도 알려줘야겠다" 싶은 게 생겨.

🔌 MCP 연결하기

MCP(Model Context Protocol)로 외부 도구를 AI에 직접 연결해. 연결하면 AI가 실시간 데이터를 조회할 수 있어.

연결 방법

Claude Code 대화 중에 자연어로 요청하면 돼:

"다우오피스 MCP 연결해줘" "Teams MCP 연결해줘" "Notion MCP 연결해줘"

연결 후 가능한 것

MCP할 수 있는 것
다우오피스메일 확인/검색, 일정 조회, 캘린더 파싱
Teams채널 메시지 읽기/보내기, 멘션 확인, 파일 공유
Notion페이지 검색/생성/수정, 데이터베이스 조회

⚠️ MCP 연결 시 인증이 필요해. 각 서비스의 API 키나 OAuth 설정을 미리 준비해두자.

⚡ 스킬 만들기

반복 업무를 /명령어 하나로 실행할 수 있게 만드는 거야.

SKILL.md 기본 구조

# /weekly-report 주간 업무 보고서를 자동으로 생성한다. ## 트리거 - "주간보고", "위클리 리포트", "/weekly-report" ## 실행 단계 1. 이번 주 완료한 업무를 정리한다 2. 다음 주 계획을 작성한다 3. 보고서 포맷에 맞춰 출력한다 ## 출력 형식 - 마크다운 표 형태 - 주요 성과 / 이슈 / 다음 주 계획

스킬 파일 위치

.claude/skills/[스킬이름]/SKILL.md

💡 좋은 스킬을 만들었으면 스킬 허브에 등록해서 동료와 공유하자!

🔗 워크플로우 구성

스킬 A의 출력이 스킬 B의 입력이 되는 멀티스텝 흐름을 만드는 거야.

예시: 데일리 브리핑 워크플로우

Step 1: 메일 확인 (다우오피스 MCP) ↓ 안 읽은 메일 목록 Step 2: 일정 파싱 (다우오피스 MCP) ↓ 오늘 미팅 리스트 Step 3: 팀즈 멘션 확인 (Teams MCP) ↓ 답장 필요한 메시지 Step 4: 투두 생성 (종합) → 오늘의 할 일 체크리스트

워크플로우 설계 팁

  • 2~3단계로 시작해. 너무 복잡하면 디버깅이 어려워.
  • 각 단계의 입력/출력을 명확하게 정의해.
  • 실패 시 대안(fallback)을 생각해둬.

🤖 에이전트 위임

스킬은 "이거 해줘". 에이전트는 "이 목표를 달성해줘, 방법은 네가 정해."

좋은 에이전트 지시문 예시

목표: 이번 주 금요일까지 고객 A사 미팅 준비 완료 해야 할 것: - CRM에서 A사 최근 활동 이력 조회 - 이전 미팅 기록 요약 - 제안할 사항 3가지 정리 - 브리핑 문서 1장으로 만들기 제약: - 내부 정보만 사용 - 본인 확인 후 최종 발송

위임 가능한 업무 체크리스트

  • ✅ 데이터 수집 + 정리 + 보고서 작성
  • ✅ 정기적인 모니터링 + 알림
  • ✅ 여러 소스를 조합한 분석
  • ❌ 최종 의사결정 (사람이 해야 해)
  • ❌ 외부 발송/게시 (확인 후에)

🧠 Memory 시스템

Memory는 대화 간 맥락을 기억하게 해주는 시스템이야. 매번 처음부터 설명할 필요가 없어져.

메모리 저장하기

"이거 기억해: 보고서는 항상 마크다운 표로 만들어줘" "기억해둬: 내 팀장은 김철수고, CC에 항상 넣어야 해"

메모리 종류

종류용도예시
User내 역할/선호도"난 마케팅팀이야"
Feedback교정/지시"요약은 3줄 이내로"
Project프로젝트 맥락"이번 분기 목표는 X"
Reference외부 정보 위치"버그는 Linear에서 추적"
